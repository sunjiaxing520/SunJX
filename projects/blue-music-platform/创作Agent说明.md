# 创作 Agent 说明

## 一句话解释

创作 Agent 不是“凭空变出一首歌”的黑箱，而是一个创作助理。

它负责把用户输入、榜单分析结果和音乐创作要求整理成一份清楚的创作素材，然后交给 Suno 这类音乐生成工具执行。

## 为什么要整理素材

如果只输入一句：

```text
帮我生成一首好听的歌
```

结果会很不可控。

所以平台要把创作要求拆成更清楚的字段，让生成结果更稳定。

## 基础版建议收集的信息

### 1. 歌曲标题

原意：这首歌叫什么名字。

怎么讲：相当于给作品起名，方便后续保存、查找和展示。

### 2. 创作类型

原意：选择带人声，还是纯音乐。

怎么讲：如果是带人声，就需要歌词；如果是纯音乐，可以不填歌词。

### 3. 歌词

原意：用户自己写歌词，或者让 AI 辅助生成歌词。

怎么讲：Suno 生成带人声歌曲时，歌词是很核心的素材。没有歌词也可以生成，但可控性会下降。

### 4. 音乐风格

原意：流行、摇滚、电子、民谣、R&B、说唱等。

怎么讲：告诉工具这首歌大概是什么类型。比如想要“流行情歌”还是“电子舞曲”。

### 5. 情绪氛围

原意：欢快、悲伤、舒缓、激昂、治愈、热血等。

怎么讲：同样是流行歌，情绪不同，生成结果会差很多。

### 6. 节奏速度

原意：慢速、中速、快速，或者 BPM 范围。

怎么讲：客户不懂 BPM 也没关系，可以说“慢一点”“适合跳舞”“有冲击力”。

### 7. 人声要求

原意：男声、女声、不指定。

怎么讲：如果用户对声音有偏好，可以提前说明。

### 8. 补充要求

原意：用户自由描述其它要求。

怎么讲：比如“适合短视频开头”“适合夜晚开车听”“副歌要抓耳”。

## 和榜单分析的关系

榜单分析结果不会直接原样丢给 Suno。

平台会把分析结果转成创作参考，例如：

```text
最近热歌偏中速、流行、情绪治愈、女声较多。
```

然后创作 Agent 再整理成：

```text
创作一首中速流行歌曲，女声，情绪治愈，副歌旋律抓耳，适合短视频传播。
```

## 对客户的讲法

可以这样说：

```text
Suno 是负责生成音乐的工具，我们的平台负责前面的整理和管理。
用户不需要自己研究怎么写复杂提示词，只要填写标题、歌词、风格、情绪、节奏、人声等中文信息，创作 Agent 会帮他整理成适合 Suno 执行的创作任务。
```

## 开发注意

- 页面上尽量显示中文字段。
- 后端可以保留英文接口字段，但不要直接暴露给客户。
- 创作工具要做适配层，避免以后换工具时重写整个业务。
- 基础版先实现单次创作任务和任务状态。
- 完整版再做在线试听、多版本管理和版本回退。
