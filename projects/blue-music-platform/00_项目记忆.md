## 三、核心功能模块

### M1：项目基础架构
- 前后端项目脚手架搭建
- PostgreSQL 数据库表结构设计
- Redis 缓存配置
- Docker Compose 编排文件
- 开发/测试/生产三套环境配置

### M2：用户与权限系统
- 注册/登录（邮箱 + 密码）
- JWT 鉴权（access_token + refresh_token）
- RBAC 权限模型：角色 → 权限 → 用户
- 管理员可创建/编辑/删除子账号
- 接口级权限中间件

### M3：Agent 热插拔框架（核心）
- Agent 配置表设计：agent_type / model_name / api_key(加密) / temperature / status
- 每个 Agent 独立配置面板（前端）
- 运行时动态加载/卸载，不重启服务
- 模型路由层：统一接口适配 OpenAI / Claude / 文心等
- API Key 加密存储（AES-256）

### M4：管理后台 Dashboard
- 平台概览：Agent 数量、用户数、榜单更新时间
- Agent 运行状态：在线/离线、调用次数、错误率
- 操作日志审计：谁在什么时间做了什么操作
- 下属使用统计：按人、按 Agent 统计

### M5：爬虫 Agent
- 爬取 QQ音乐 热歌榜 Top500
- 定时任务（每天/每周，可配置）
- 反反爬策略：UA 轮换、请求间隔、代理 IP 接入
- 数据清洗：去重、格式标准化
- 存储字段：歌曲名、歌手、专辑、播放量、排名、爬取时间

### M6：分析 Agent
- 调用 LLM 对爬取的歌曲进行风格分析
- 输出标签：曲风（流行/摇滚/电子/民谣/R&B/嘻哈）、情绪（欢快/悲伤/激昂/舒缓）、BPM 范围、语种
- 关键词提取：从歌词、评论中提取核心关键词
- 生成风格趋势报告

### M7：创作 Agent
- 对接 Suno API
- 根据分析结果自动生成创作 Prompt
- 支持批量创作（一次生成 N 首）
- 音乐文件下载、元数据存储
- 创作历史：可回放、可删除

### M8：权限分发（下属管理）
- 管理员创建角色（如：爬虫员、分析师、创作者）
- 为角色绑定 Agent 权限
- 将角色分配给下属
- 下属登录后只能看到和使用被授权的 Agent

## 四、数据库核心表设计（概要）

```
users          — 用户表（id, email, password_hash, role_id, created_at）
roles          — 角色表（id, name, description）
role_agent_permissions — 角色-Agent权限关联（role_id, agent_id）
agents         — Agent实例表（id, agent_type, model_name, api_key_encrypted, temperature, status, owner_id）
crawled_songs  — 爬取歌曲表（id, song_name, artist, album, play_count, rank, platform, crawled_at）
song_analysis  — 分析结果表（id, song_id, style_tags, emotion, bpm, keywords, analyzed_at）
creations      — 创作记录表（id, agent_id, prompt, audio_url, style_ref, created_at）
audit_logs     — 操作日志表（id, user_id, action, target, ip, created_at）
```
