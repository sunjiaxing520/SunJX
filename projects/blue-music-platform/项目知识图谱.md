# 蓝乐项目知识图谱

这份文档通过文件之间的引用关系，展示项目代码如何连接和运行。

图中约定：

- 实线：当前已经存在并运行的关系。
- 虚线：后续计划实现的模块。
- 箭头：前面的文件使用、调用或依赖后面的文件。

## 1. 当前文件引用图

```mermaid
flowchart TD
    Main["main.py<br/>创建并组装整个后端"]
    Config["core/config.py<br/>项目配置"]
    ApiRouter["api/v1/router.py<br/>总接口集合"]
    HealthRoute["routes/health.py<br/>健康检查接口"]
    Database["core/database.py<br/>数据库连接和 Session"]
    HealthService["services/health.py<br/>数据库检查逻辑"]
    HealthSchema["schemas/health.py<br/>健康检查返回格式"]
    TestConfig["tests/conftest.py<br/>创建测试客户端"]
    HealthTest["tests/test_health.py<br/>验证健康检查行为"]
    Alembic["alembic/env.py<br/>数据库结构版本环境"]
    Metadata["Base.metadata<br/>全部数据库表结构信息"]

    Main -->|读取项目名称、版本和地址前缀| Config
    Main -->|安装总接口集合| ApiRouter
    ApiRouter -->|收集健康检查接口| HealthRoute
    HealthRoute -->|读取版本和环境| Config
    HealthRoute -->|通过 get_db 获取 Session| Database
    HealthRoute -->|调用数据库检查函数| HealthService
    HealthRoute -->|规定成功返回格式| HealthSchema
    HealthService -->|使用 Session 执行 SELECT 1| Database
    Database -->|创建并持有| Metadata
    Alembic -->|读取表结构| Metadata
    Alembic -->|读取数据库地址| Config
    TestConfig -->|导入并包装| Main
    HealthTest -->|发送测试请求| TestConfig
    HealthTest -->|临时替换 get_db| Database
```

### 数据库健康检查

```mermaid
flowchart LR
    Browser["浏览器或前端"]
    Address["GET /api/v1/health/database"]
    Route["database_health_check()"]
    Depends["Depends(get_db)"]
    GetDB["get_db()"]
    Session["Session"]
    Service["is_database_available()"]
    SQL["SELECT 1"]
    DB["PostgreSQL"]

    Browser --> Address --> Route
    Route --> Depends --> GetDB --> Session
    Route --> Service
    Service --> Session --> SQL --> DB
    DB -->|执行成功| Success
    DB -->|发生 SQLAlchemyError| Failure
    Success --> Close
    Failure --> Close
```

## 8. 如何使用这份图谱

看到一个陌生文件时：

1. 先在“当前文件引用图”中找到它。
2. 查看谁指向它，理解谁会使用它。
3. 查看它指向谁，理解它依赖什么。
4. 再打开源代码逐行阅读。
5. 阅读相应测试，确认代码承诺的行为。

后续每增加登录、用户、榜单、分析、作词或创作模块，都要同时更新这份知识图谱。
