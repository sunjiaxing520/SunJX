# 蓝乐 AI 音乐创作平台 —— 需求规格说明书（精简版）v3.0

## 0. 项目一句话

蓝乐 AI 音乐创作平台是一套面向音乐创作者的 AI 工作台。

它先采集酷狗热歌榜数据，再分析热门歌曲特点，然后由作词 Agent 辅助写歌词，最后由创作 Agent 整理素材并提交给 Suno 生成音乐。

核心链路：

```text
酷狗榜单采集 -> 歌曲风格分析 -> AI 作词 -> Suno 创作 -> 结果展示
```

## 1. 当前交付边界

首期目标是做一个固定、清楚、可演示、可验收的版本。

首期先做：

```text
人操作多个固定 Agent
```

暂不做：

```text
用户自定义工作链
总调度 Agent 自动拆任务
多个增强 Agent 自动协作
复杂长期记忆系统
```

这些放入后续增强项目，等基础项目跑通后再单独报价。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 管理员 | 管理用户、查看数据、配置 Agent、查看任务 |
| 普通用户 | 使用榜单、分析、作词、创作等功能 |

## 3. Agent 设计

首期包含 4 个核心 Agent。

| Agent | 作用 | 首期定位 |
|---|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜数据 | 自动采集机器人 |
| 音乐分析 Agent | 分析歌曲风格、情绪、关键词 | AI 分析助手 |
| 作词 Agent | 根据主题、风格、用户偏好生成歌词 | 带基础记忆的 AI 作词助手 |
| Suno 创作 Agent | 整理素材并提交给 Suno | 自动提交和结果回收机器人 |

说明：

- “Agent”不一定都代表复杂 AI。
- 有些 Agent 是自动化机器人，例如爬虫 Agent、Suno 创作 Agent。
- 作词 Agent 需要 AI 介入，也需要基础记忆，因为歌词风格和客户偏好会影响结果。

## 4. 功能需求

### FR-001 用户登录

原文要求：用户可以注册、登录，系统使用账号密码和令牌保护接口。

精简后要求：

- 支持管理员登录。
- 支持普通用户登录。
- 密码加密存储。
- 登录后才能使用后台功能。

验收标准：

- 能成功登录后台。
- 错误密码不能登录。
- 登录后能看到工作台。

### FR-002 权限管理

原文要求：系统支持管理员和普通用户角色。

精简后要求：

- 管理员可以管理全部功能。
- 普通用户只能使用被允许的功能。
- 后端接口要做权限校验。

验收标准：

- 普通用户不能访问管理员功能。
- 无权限访问时返回错误提示。

### FR-003 酷狗榜单采集

原文中曾出现 QQ 音乐和酷狗音乐不一致。现已确认：

```text
首期先爬酷狗热歌榜。
```

功能要求：

- 支持手动触发酷狗热歌榜采集。
- 采集歌曲名、歌手、排名、热度、封面、采集时间。
- 数据入库前去重。
- 失败时记录错误信息。

验收标准：

- 后台点击按钮后能创建采集任务。
- 采集结果能在列表中展示。
- 同一榜单同一歌曲不重复入库。

### FR-004 音乐分析

功能要求：

- 对采集到的歌曲做基础分析。
- 输出风格、情绪、关键词、节奏倾向。
- 分析结果保存到系统中。

验收标准：

- 用户能选择歌曲并执行分析。
- 分析结果能在后台查看。
- 分析结果能作为作词和创作参考。

### FR-005 作词 Agent

这是本次新增的核心功能。

功能定位：

```text
帮助用户用 AI 写歌词。
```

用户输入：

- 歌曲主题。
- 音乐风格。
- 情绪氛围。
- 目标场景，例如短视频、表白、治愈、夜晚、国风等。
- 想要的人声方向。
- 可选参考歌词或关键词。

Agent 记忆：

首期先做“基础记忆”，不做复杂长期记忆系统。

基础记忆包含：

- 用户常用主题。
- 用户偏好的歌词风格。
- 常用表达方式。
- 不想要的词或风格。
- 已生成歌词记录。

功能要求：

- 根据用户输入生成歌词。
- 支持重新生成。
- 支持局部修改，例如重写副歌、改成更口语、更伤感、更抓耳。
- 生成结果可一键送入 Suno 创作 Agent。

验收标准：

- 用户能输入主题并生成歌词。
- 用户能保存满意歌词。
- 用户能把歌词带入创作任务。
- 作词 Agent 能读取用户基础偏好作为参考。

### FR-006 Suno 创作 Agent

功能定位：

```text
把创作素材整理成 Suno 可执行任务。
```

注意：

首期不自研音乐生成模型，音乐生成能力暂时由 Suno 提供。

Suno 创作 Agent 负责：

- 整理歌曲标题。
- 整理歌词。
- 整理音乐风格。
- 整理情绪氛围。
- 整理节奏速度。
- 整理人声要求。
- 提交给 Suno。
- 跟踪任务状态。
- 保存生成结果。

验收标准：

- 用户能创建 Suno 创作任务。
- 系统能展示任务状态：排队中、执行中、完成、失败。
- 结果链接能保存到平台。

### FR-007 Dashboard 工作台

功能要求：

- 展示今日采集歌曲数。
- 展示已分析歌曲数。
- 展示作词任务数。
- 展示创作任务数。
- 展示 Agent 状态。

验收标准：

- 用户登录后能看到清晰工作台。
- 可以查看榜单、分析、歌词、创作任务。

### FR-008 演示前端

功能要求：

- 提供一个可给客户演示的前端 demo。
- 视觉风格采用高级、精致、音乐工具感。
- 功能展示包括：榜单、二创、作词、Suno 创作、Agent、后续工作链设想。

验收标准：

- 打开 demo 后客户能看懂平台流程。
- 页面能体现“人操作多个 Agent”的首期模式。
- 页面能体现“后续工作链/总调度 Agent”是增强功能。

## 5. 后续增强功能

以下功能不纳入首期，可作为后续加费项目。

### 5.1 自定义 Agent 工作链

用户可以自己选择多个 Agent 组成流程。

示例：

```text
酷狗榜单采集 -> 爆款趋势分析 -> 作词 Agent -> Suno 创作 Agent
```

价值：

- 客户能自己搭流程。
- 不同 Agent 可以有不同风格和偏好。
- 平台从固定工具升级成可配置工作台。

### 5.2 增强记忆系统

首期作词 Agent 只做基础偏好记忆。

后续可升级为更完整记忆系统：

- 长期记录客户创作偏好。
- 根据历史反馈调整作词风格。
- 为不同客户建立不同创作档案。
- 支持多个作词 Agent 拥有不同记忆。

### 5.3 总调度 Agent

用户只描述目标，总调度 Agent 自动拆任务。

示例：

```text
用户说：参考最近热歌，做一首适合短视频传播的治愈系女声歌。
```

系统自动：

```text
采集榜单 -> 分析趋势 -> 生成歌词 -> 整理 Suno 创作任务 -> 保存结果
```

这是高级功能，不放入首期。


## 8. 版本范围

### 基础版

基础版目标是跑通完整闭环：

```text
登录 -> 酷狗榜单 -> 音乐分析 -> AI 作词 -> Suno 创作 -> Dashboard 展示
```

### 完整版

完整版在基础版上增加：

- 多数据源爬取。
- 定时任务。
- 更丰富图表。
- 在线试听。
- 创作版本管理。
- 操作日志。
- 数据备份。

### 后续增强

后续增强单独评估：

- 自定义工作链。
- 多个作词 Agent。
- 多个创作 Agent。
- Agent 长期记忆。
- 总调度 Agent。
