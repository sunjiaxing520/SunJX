## 1. 引言

### 1.1 项目背景

蓝乐 AI 音乐创作平台旨在利用 AI Agent 技术，构建一套从音乐数据采集、风格分析到辅助创作的智能化平台。平台采用 Agent 热插拔框架，支持灵活扩展不同能力的智能体，为音乐创作者提供数据驱动、AI 辅助的一站式工作环境。

### 1.2 产品定位

面向音乐创作者、制作人和爱好者的 AI 辅助创作平台，通过智能爬虫采集音乐榜单数据，结合多维分析 Agent 解析音乐特征，辅助用户完成音乐创作。

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI (Python 3.10+) | RESTful API，异步高性能 |
| 前端框架 | React 18 + TypeScript | SPA 单页应用 |
| 数据库 | PostgreSQL 15+ | 主数据存储 |
| 缓存 | Redis 7+ | 会话管理、任务队列、热数据缓存 |
| 容器化 | Docker + Docker Compose | 一键部署，环境隔离 |

### 1.4 术语与缩写

| 术语 | 说明 |
|------|------|
| Agent | 可插拔的智能体模块，独立完成特定任务 |
| RBAC | 基于角色的访问控制 (Role-Based Access Control) |
| Hot-plug | 热插拔，无需重启系统即可动态加载/卸载 Agent |
| 【8K】 | 完整版 8K 报价方案专有功能标记 |

## 3. 功能需求

---

### 3.1 用户管理模块

#### FR-0001 用户注册

- **描述**：新用户通过邮箱或用户名注册平台账号。
- **输入**：用户名（3-20 字符）、邮箱、密码（8-32 字符，含大小写字母+数字）。
- **输出**：注册成功返回用户 ID，失败返回错误信息。
- **验收标准**：
  1. 用户名/邮箱唯一性校验通过后创建账号。
  2. 密码经 bcrypt 哈希后存入数据库，禁止明文存储。
  3. 注册成功后自动分配 `user` 角色。

#### FR-0002 用户登录

- **描述**：已注册用户通过用户名/邮箱 + 密码登录。
- **输入**：账号标识（用户名或邮箱）、密码。
- **输出**：登录成功返回 JWT Access Token（2h 有效）+ Refresh Token（7d 有效）。
- **验收标准**：
  1. 连续 5 次登录失败后锁定账号 15 分钟。
  2. Token 过期后 Refresh Token 可刷新 Access Token。
  3. 登录日志记录 IP 和时间戳。

#### FR-0003 个人信息管理

- **描述**：用户查看和编辑个人基本信息。
- **输入**：昵称、头像 URL、个人简介（可选）。
- **输出**：更新后的用户信息。
- **验收标准**：
  1. 仅允许编辑本人信息。
  2. 管理员可编辑任意用户信息。
  3. 修改邮箱/密码需验证当前密码。

---

### 3.2 权限管理 (RBAC)

#### FR-0004 角色定义

- **描述**：系统预置 `admin` 和 `user` 两种角色。
- **验收标准**：
  1. `admin` 拥有全部 API 端点访问权限。
  2. `user` 仅拥有业务操作和 Dashboard 查看权限。
  3. 角色定义不可删除，权限集合可配置。

#### FR-0005 权限分配与校验

- **描述**：API 网关层基于 JWT 中的角色字段进行权限校验。
- **验收标准**：
  1. 每个 API 端点在路由注册时声明所需角色。
  2. 无权限请求返回 `403 Forbidden`。
  3. 过期 Token 返回 `401 Unauthorized`。

#### FR-0006 用户角色管理（管理员）

- **描述**：管理员可查看用户列表，修改用户角色。
- **验收标准**：
  1. 支持按角色筛选用户列表。
  2. 角色变更后当前会话 Token 自动失效。
  3. 角色变更记录写入操作日志。

---

### 3.3 Agent 热插拔框架

#### FR-0007 Agent 注册与发现

- **描述**：Agent 启动时向框架注册自身元信息（名称、类型、能力描述、槽位占用）。
- **输入**：Agent 元信息 JSON。
- **输出**：注册成功返回 Agent ID。
- **验收标准**：
  1. 基础版支持最多 3 个槽位。
  2. 支持最多 5 个槽位。
  3. Agent 注册时校验槽位是否已满。

#### FR-0008 Agent 配置管理（基础版）

- **描述**：管理员通过配置界面手动启用/禁用/配置 Agent。
- **验收标准**：
  1. Agent 状态变更需手动保存生效。
  2. 配置变更后需手动重启对应 Agent。

#### FR-0009 Agent 热插拔 【8K】

- **描述**：【8K】支持 Agent 运行中动态加载与卸载，无需重启系统。
- **验收标准**：
  1. Agent 加载后 10 秒内完成注册并开始工作。
  2. Agent 卸载时优雅关闭（等待当前任务完成或超时 30s 强制终止）。
  3. 热插拔操作不影响其他 Agent 正常运行。
  4. 热插拔事件记录到操作日志。

#### FR-0010 Agent 心跳与健康检查

- **描述**：框架定期检测 Agent 心跳，异常 Agent 自动标记不可用。
- **验收标准**：
  1. 心跳间隔 10 秒。
  2. 连续 3 次心跳丢失判定为异常。
  3. Dashboard 实时显示各 Agent 运行状态。

---

### 3.4 爬虫 Agent

#### FR-0011 QQ 音乐热歌榜爬取（基础版）

- **描述**：爬虫 Agent 定时抓取 QQ 音乐热歌榜数据。
- **数据字段**：歌曲名、歌手、排名、热度值、封面 URL、爬取时间。
- **验收标准**：
  1. 单次爬取成功率 ≥ 95%。
  2. 数据去重：同榜同歌曲不重复入库。
  3. 爬取频率：手动触发，最小间隔 10 分钟。
  4. 异常重试 3 次，失败后记录日志。

#### FR-0012 多源爬取 【8K】

- **描述**：【8K】爬虫 Agent 支持网易云音乐和酷狗音乐多数据源爬取。
- **数据字段**：在 FR-0011 基础上新增数据源标识字段。
- **验收标准**：
  1. 网易云音乐热歌榜数据获取成功率 ≥ 90%。
  2. 酷狗音乐热歌榜数据获取成功率 ≥ 90%。
  3. 多源数据统一格式存储，来源字段区分。

#### FR-0013 定时调度 【8K】

- **描述**：【8K】支持 Cron 表达式配置定时爬取任务。
- **验收标准**：
  1. 默认每日 02:00 自动执行全量爬取。
  2. 支持自定义 Cron 表达式。
  3. 定时任务执行状态可在 Dashboard 查看。
  4. 同一数据源同时仅允许一个爬取任务运行。

### 3.5 分析 Agent

#### FR-0014 基础风格分类（基础版）

- **描述**：分析 Agent 对爬取的音乐数据进行风格分类。
- **分类类别**：流行 (Pop)、摇滚 (Rock)、电子 (Electronic)、民谣 (Folk)、R&B。
- **验收标准**：
  1. 分类置信度 ≥ 0.6 的结果标记为有效。
  2. 支持对单首歌曲和批量歌曲进行分析。
  3. 分析结果包含风格标签 + 置信度分数。

#### FR-0015 情绪分析 【8K】

- **描述**：【8K】分析 Agent 对音乐进行情绪维度分析。
- **情绪维度**：愉悦度 (Valence)、唤醒度 (Arousal)、能量水平。
- **验收标准**：
  1. 每个维度输出 0.0-1.0 归一化分数。
  2. 结合三值输出情绪标签（如"激昂"、"舒缓"、"忧郁"等）。
  3. 支持按情绪维度筛选和排序音乐。

#### FR-0016 编曲分析 【8K】

- **描述**：【8K】分析 Agent 对音乐编曲结构进行分析。
- **分析维度**：BPM 估算、调性识别、乐器组成推测、段落结构识别。
- **验收标准**：
  1. BPM 估算误差 ≤ ±5 BPM。
  2. 调性识别准确率 ≥ 80%。
  3. 分析结果以结构化 JSON 输出。

#### FR-0017 和弦分析 【8K】

- **描述**：【8K】分析 Agent 对音乐和弦进行进行识别。
- **验收标准**：
  1. 输出和弦进程序列（如 I-V-vi-IV）。
  2. 识别常用和弦进行模式。
  3. 结果可与创作 Agent 联动，作为创作参考输入。

---

### 3.6 创作 Agent

#### FR-0018 外部创作网站对接

- **描述**：创作 Agent 通过 Web 自动化操控外部在线音乐创作工具，执行用户指定的创作任务。
- **验收标准**：
  1. 支持至少 1 个外部创作网站（如 Soundtrap / BandLab / AudioTool）。
  2. 用户提交创作参数（风格、BPM、调性、情绪描述）后 Agent 自动执行。
  3. 创作任务状态实时反馈（排队中 → 执行中 → 完成/失败）。
  4. 创作产物（MIDI/音频链接）保存并可在平台内预览。

#### FR-0019 创作任务管理

- **描述**：用户可查看、取消、重试创作任务。
- **验收标准**：
  1. 任务列表支持按状态筛选。
  2. 取消操作立即终止 Agent 执行。
  3. 历史任务保留 30 天。

### 3.7 Dashboard

#### FR-0020 统计卡片（基础版）

- **描述**：Dashboard 首页展示关键统计指标卡片。
- **统计项**：今日爬取歌曲数、分析完成数、创作任务数、活跃 Agent 数。
- **验收标准**：
  1. 数据每 30 秒自动刷新。
  2. 卡片支持点击跳转至详情页。

#### FR-0021 基础数据表格（基础版）

- **描述**：Dashboard 以表格形式展示爬取数据、分析结果、任务列表。
- **验收标准**：
  1. 表格支持分页（默认 20 条/页）。
  2. 支持按列排序（歌曲名、时间、热度等）。
  3. 支持关键字搜索过滤。

#### FR-0022 ECharts 可视化图表 【8K】

- **描述**：【8K】Dashboard 集成 ECharts，提供多维度数据可视化。
- **图表类型**：
  - 饼图：风格分布占比
  - 柱状图：各数据源歌曲数量对比
  - 趋势线：爬取量/分析量时间趋势
  - 热力图：情绪维度分布
- **验收标准**：
  1. 图表支持时间范围筛选（7天/30天/自定义）。
  2. 图表数据与后端 API 实时同步。
  3. 图表支持导出为 PNG 图片。

#### FR-0028 音乐在线试听 【8K】

- **描述**：【8K】生成的音乐可直接在网页端播放试听，基础版仅提供文件名列表。
- **验收标准**：
  1. 支持 Web Audio API 在线播放，含播放/暂停/进度控制。
  2. 支持 MP3/WAV 格式音频播放。
  3. 播放列表与创作任务结果联动。

#### FR-0029 风格对比面板 【8K】

- **描述**：【8K】多首歌曲的风格特征并排对比，ECharts 雷达图可视化，基础版仅支持单首歌曲查看。
- **验收标准**：
  1. 支持同时选择 2-6 首歌曲进行对比。
  2. 雷达图展示风格/情绪/能量等多维度对比。
  3. 对比结果支持截图导出。

#### FR-0030 导出报表 【8K】

- **描述**：【8K】分析结果一键导出为 PDF 或 Excel 报表，基础版仅支持在线查看。
- **验收标准**：
  1. 支持导出格式：PDF、Excel (.xlsx)。
  2. 导出内容包含分析图表与数据明细。
  3. 服务端渲染生成，导出文件可下载。

### 3.8 系统备份与回档 【8K】

#### FR-0023 定时备份 【8K】

- **描述**：【8K】系统支持 PostgreSQL 数据库定时全量备份。
- **验收标准**：
  1. 默认每日 03:00 自动备份。
  2. 备份文件以日期命名：`backup_YYYYMMDD_HHmmss.sql.gz`。
  3. 备份文件使用 AES-256 加密存储。
  4. 保留最近 30 天的备份文件，自动清理过期备份。

#### FR-0024 手动备份 【8K】

- **描述**：【8K】管理员可在管理界面手动触发即时备份。
- **验收标准**：
  1. 手动备份立即执行，不受定时任务影响。
  2. 备份完成后提供文件下载链接。

#### FR-0025 一键回档 【8K】

- **描述**：【8K】管理员选择历史备份文件执行数据回档。
- **验收标准**：
  1. 回档前自动生成当前状态快照备份。
  2. 回档过程平台进入维护模式，拒绝新请求。
  3. 回档完成后自动恢复服务，全程 ≤ 10 分钟。
  4. 回档操作需二次确认并记录到操作日志。

---

### 3.9 操作日志 【8K】

#### FR-0026 用户操作全量记录 【8K】

- **描述**：【8K】系统记录所有用户的增删改操作日志。
- **记录字段**：操作人 ID、操作类型、目标资源、操作详情 (JSON)、IP 地址、时间戳。
- **验收标准**：
  1. 日志写入异步完成，不影响主业务响应时间。
  2. 日志表按月分表，保证查询性能。
  3. 日志保留周期 180 天，超期自动归档。

#### FR-0027 审计查询 【8K】

- **描述**：【8K】管理员可通过管理界面查询和筛选操作日志。
- **验收标准**：
  1. 支持按操作人、操作类型、时间范围筛选。
  2. 支持关键字搜索操作详情。
  3. 查询结果支持导出为 CSV 文件。

## 5. API 接口设计

### 5.1 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.2 认证组 (5 个接口)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/auth/register | 用户注册 |

| POST | /api/v1/auth/refresh | 刷新 Token |
| POST | /api/v1/auth/logout | 登出 |
| GET | /api/v1/auth/me | 获取当前用户信息 |

  id, user_id, action_type, resource_type, resource_id, detail(JSON), ip_address, created_at

  id, file_name, file_size, backup_type(auto/manual), encrypted, created_at
