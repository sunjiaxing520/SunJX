# 蓝乐项目中文代码地图

这份地图放在项目最外层，用来帮助初次接触项目的人理解每个文件夹和主要文件的用途。

代码目录继续使用英文，是为了兼容 Python 导入、Docker、Linux 服务器和第三方开发工具；中文含义以本地图为准。

## 1. 从哪里开始看

第一次阅读建议按照以下顺序：

1. `中文代码地图.md`：先认识整个项目。
2. `项目知识图谱.md`：查看文件如何相互引用和运行。
3. `README.md`：理解产品目标和开发顺序。
4. `backend/app/main.py`：理解后端如何启动。
5. `backend/app/api/v1/router.py`：理解接口如何集中注册。
6. `backend/app/api/v1/routes/health.py`：阅读第一个真实接口。
7. `backend/app/services/health.py`：理解业务逻辑为什么单独存放。
8. `backend/tests/test_health.py`：理解测试如何证明接口正常。
9. `docs/03-implementation/P1_数据库基础讲解.md`：结合中文讲解复习代码。
10. `frontend/src/App.tsx`：理解前端页面和权限路由如何组合。

## 4. 后端核心代码

```text
backend/app/
├── api/                 接口入口层
├── core/                项目基础设施
├── models/              数据库表模型
├── schemas/             请求和响应格式
├── services/            业务逻辑
└── main.py              FastAPI 启动入口
```

### app/main.py：启动入口

主要负责：

- 创建 FastAPI 应用。
- 注册 CORS 中间件。
- 挂载 API 总路由。
- 提供根路径接口。

它应该只负责组装应用，不应该堆放登录、数据库或 Agent 业务代码。

### app/api：接口入口层

```text
api/v1/router.py             汇总 v1 版本的全部接口
api/v1/routes/health.py      健康检查接口
api/v1/routes/auth.py        登录和当前用户接口
api/v1/routes/dashboard.py   工作台汇总接口
api/v1/routes/users.py       成员账号和Agent权限管理
api/dependencies.py          Token、当前用户和权限依赖
```

Route 的职责是：

- 接收 HTTP 请求。
- 校验请求参数。
- 调用 Service。
- 决定 HTTP 状态码。
- 返回响应。

Route 不应该直接堆积复杂业务和大量数据库操作。

### app/core：基础设施

```text
core/config.py       项目配置和环境变量
core/database.py     数据库 Engine、Session 和依赖注入
core/security.py     Argon2密码和JWT
core/logging.py      本地与云端结构化日志
core/request_context.py  请求编号
```

安全、日志、异常和请求追踪等全局基础能力放在这里。

### app/models：数据库表模型

当前已有：

```text
models/user.py           用户表
models/agent_permission.py  成员Agent权限表
```

以后可能出现：

```text
models/song.py           榜单歌曲表
models/analysis.py       音乐分析结果表
models/lyrics.py         作词任务和歌词版本表
models/creation.py       音乐创作任务表
```

Model 描述数据库中有什么表、字段、约束和关联关系。

### app/schemas：数据格式

当前已有：

```text
schemas/health.py    健康检查响应格式
schemas/auth.py      登录请求和Token响应
schemas/dashboard.py 工作台指标和Agent状态响应
schemas/user.py      用户管理请求和响应
```

Schema 使用 Pydantic 定义：

- 前端允许传入什么字段。
- 字段是什么类型。
- 后端承诺返回什么结构。

Schema 不是数据库表。

### app/services：业务逻辑

当前已有：

```text
services/health.py   检查数据库能否执行查询
services/auth.py     账号与密码认证
services/dashboard.py 组织当前用户可见的工作台数据
services/users.py    用户和Agent权限管理
```

以后登录校验、榜单去重、AI 调用和任务状态处理都会优先放在 Service，而不是 Route。

## 8. 一次请求如何经过代码

以数据库健康检查为例：

```text
浏览器或前端
-> GET /api/v1/health/database
-> app/main.py 中挂载的总路由
-> app/api/v1/router.py
-> app/api/v1/routes/health.py
-> Depends(get_db)
-> app/core/database.py 创建 Session
-> app/services/health.py 执行 SELECT 1
-> app/schemas/health.py 校验响应
-> 返回 HTTP 200 或 HTTP 503
-> Session 被关闭
```

## 9. 看见新文件时怎么判断

可以先根据路径判断职责：

| 路径中包含 | 中文含义 | 先问什么 |
|---|---|---|
| `routes` | 接口入口 | 它接收哪个 URL 和 HTTP 方法？ |
| `schemas` | 数据格式 | 前端传什么，后端返回什么？ |
| `services` | 业务逻辑 | 它具体处理什么规则？ |
| `models` | 数据库表 | 数据保存在哪些字段？ |
| `core` | 基础设施 | 哪些模块共同依赖它？ |
| `tests` | 自动化验证 | 它证明了哪种行为？ |
| `alembic` | 数据库迁移 | 这次表结构发生了什么变化？ |

后续每新增一个重要目录或文件，都需要同步更新这份地图。
