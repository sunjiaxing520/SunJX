# 蓝乐 AI 音乐创作平台 —— 需求规格说明书（精简版）v3.7

## 0. 项目一句话

蓝乐 AI 音乐创作平台是一套面向音乐创作者的 AI 工作台。

它先采集酷狗热歌榜数据，再分析热门歌曲特点，然后由作词 Agent 辅助写歌词，最后通过官方音乐生成 API 生成音乐。

核心链路：

```text
酷狗榜单采集 -> 榜单趋势分析 -> AI 作词 -> 音乐生成 -> 结果展示
```

## 1. 当前交付边界

当前目标是做一个清楚、可演示、可验收的线性可配置版本。

首期先做：

```text
用户选择需要的既有 Agent 步骤，系统按数据依赖顺序自动执行并传递产出
```

暂不做：

```text
拖拽式任意工作链和手工参数映射
条件分支、并行节点和循环
总调度 AI 自动理解目标和拆任务
多个增强 Agent 自动协作
复杂长期记忆系统
```

当前已支持采集、分析、作词的线性选择和模板保存；上面的高级编排能力放入后续增强项目。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 超级管理员 | 创建和停用账号、分配 Agent 权限、查看团队数据和任务 |
| 团队成员 | 使用被授权的榜单、分析、作词和创作功能 |

## 3. Agent 设计

首期包含 4 个核心 Agent，并由固定工作流统一调度。

| Agent | 作用 | 首期定位 |
|---|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜数据 | 自动采集机器人 |
| 音乐分析 Agent | 根据榜单元数据分析风格、情绪和趋势 | AI 分析助手 |
| 作词 Agent | 根据主题、风格和分析结果生成歌词 | 使用当前启用的文本模型配置 |
| 音乐创作 Agent | 整理素材并调用官方音乐生成 API | 自动提交和结果回收机器人 |

说明：

- “Agent”不一定都代表复杂 AI。
- 有些 Agent 是自动化机器人，例如爬虫 Agent、音乐创作 Agent。
- Agent 之间不直接互相调用，由工作流调度器控制固定执行顺序。
- 作词 Agent 需要 AI 介入；首期同一时刻使用一个当前配置，但超级管理员可以预存并热切换多个文本模型连接，独立记忆仍放入后续增强范围。

## 4. 功能需求

### FR-001 用户登录

原文要求：用户可以登录，系统使用账号密码和令牌保护接口。

精简后要求：

- 支持超级管理员登录。
- 支持内部团队成员登录。
- 不开放用户自助注册，由超级管理员创建和停用账号。
- 密码加密存储。
- 登录后才能使用后台功能。

验收标准：

- 能成功登录后台。
- 错误密码不能登录。
- 登录后能看到工作台。

### FR-002 权限管理

当前要求：系统支持超级管理员和内部团队成员两类角色。

精简后要求：

- 超级管理员可以管理全部功能、账号和 Agent 权限。
- 团队成员只能运行被允许的 Agent。
- 榜单、分析、歌词和创作结果默认在团队内共享。
- 后端接口要做权限校验。

验收标准：

- 团队成员不能访问超级管理员功能。
- 无权限访问时返回错误提示。

### FR-003 酷狗榜单采集

原文中曾出现 QQ 音乐和酷狗音乐不一致。现已确认：

```text
首期先爬酷狗热歌榜。
```

功能要求：

- 支持手动触发酷狗热歌榜采集。
- 采集歌曲名、歌手、排名、时长、来源链接和采集时间；热度、封面仅在公开榜单源提供时保存，缺失时不伪造。
- 数据入库前去重。
- 失败时记录错误信息。
- 按自然日保存榜单快照，默认保留最近 30 天。
- 同一天重复采集时覆盖当天快照，并把采集完成时间更新为最后一次成功采集时间；页面展示最近 15 个采集结果和最近 15 次运行记录。
- 采集运行记录支持单条和批量删除；删除运行记录不删除按日保存的榜单快照。
- 采集模块主页面只展示结果摘要；排行榜歌曲不在主页面全部展开，用户点击某个结果后再加载该快照详情。

验收标准：

- 后台点击按钮后能创建采集任务。
- 采集结果能在列表中展示，点击结果能打开对应榜单详情。
- 同一榜单同一歌曲不重复入库。
- 连续采集后能够按日期查看榜单，并支持默认 7 天趋势分析。

### FR-004 音乐分析

功能要求：

- 对采集到的榜单元数据做基础趋势分析。
- 输出风格、情绪、主题、场景、速度档位、人声、乐器、结构和 Hook 等创作方向。
- 首期不下载酷狗音频，也不承诺基于音频计算真实 BPM、调性和编曲结构。
- 分析结果保存到系统中。
- 分析记录支持单条和批量删除；删除时同步删除报告与分析收藏，但保留接口用量审计和已经生成的歌词。
- 用户可以把满意的某一份分析报告加入团队收藏夹，收藏记录保留来源任务、生成时间、模型、Token、收藏人和收藏时间。

验收标准：

- 用户能选择歌曲并执行分析。
- 分析结果能在后台查看。
- 分析结果能作为作词和创作参考。

### FR-005 作词 Agent

这是本次新增的核心功能。

功能定位：

```text
帮助用户用 AI 写歌词。
```

用户输入：

- 歌曲主题。
- 音乐风格。
- 情绪氛围。
- 目标场景，例如短视频、表白、治愈、夜晚、国风等。
- 想要的人声方向。
- 可选参考歌词或关键词。

功能要求：

- 根据用户输入生成歌词。
- 支持重新生成。
- 保存本次生成结果和错误信息。
- 作词记录最多读取最近 15 个任务，页面默认只展示最新 3 条，其余记录按需展开。
- 支持单条或勾选多条作词记录永久删除；删除任务时同步删除其歌词版本和歌词收藏，但保留接口用量审计。
- 用户可以收藏具体歌词版本；后续重新生成的新版本不会替换已经收藏的旧版本。
- 生成结果整理为统一创作方案；P7 完成后由固定工作流送入音乐创作 Agent。

验收标准：

- 用户能输入主题并生成歌词。
- 用户能保存满意歌词。
- 用户能把歌词带入创作任务。
- 用户能在团队收藏夹按分析、歌词分类查看精选产物，备注默认隐藏并可单独展开编辑。

### FR-006 音乐创作 Agent

功能定位：

```text
把创作素材整理成官方音乐生成 API 可执行的任务。
```

注意：

首期不自研音乐生成模型，音乐生成能力由正式供应商的官方 API 提供。

音乐创作 Agent 负责：

- 整理歌曲标题。
- 整理歌词。
- 整理音乐风格。
- 整理情绪氛围。
- 整理节奏速度。
- 整理人声要求。
- 提交给已配置的音乐生成供应商。
- 跟踪任务状态。
- 保存生成结果、音频格式、时长、试听地址和下载地址。
- 同一任务返回多个音频时，每个音频作为独立结果展示和操作。
- 系统提供独立“试听区”，统一罗列已经生成完成的音乐结果；点击某一首后播放该结果的完整音频。
- 生成结果可以是首段初稿，也可以是续写后的完整作品；两者都是真实音乐结果，不使用单独的“试听版”接口。
- 供应商支持续写时，用户可以在听完首段初稿后主动点击“继续续写”，由系统引用原结果创建新的续写任务。
- 试听只播放已经保存的音频，不触发音乐生成；续写是一次新的外部生成请求，必须单独展示状态、失败原因和结果。
- 续写不会覆盖原始初稿，系统保存原结果与续写结果之间的来源关系。

验收标准：

- 用户能通过固定工作流创建音乐创作任务。
- 系统能展示任务状态：排队中、执行中、完成、失败。
- 已完成的音频结果会出现在试听区，用户点击某一首即可播放，播放失败时显示明确原因。
- 对支持续写的结果，用户能在试听后主动创建续写任务；原始结果继续保留，续写结果完成后也进入试听区。
- 每个音频结果都有独立下载按钮，下载时由用户通过浏览器选择本机保存位置；浏览器不支持选择位置时使用浏览器默认下载流程。
- 音频结果能下载到客户自己的存储并在平台重新查看。

### FR-007 Dashboard 工作台

功能要求：

- 展示今日采集歌曲数。
- 展示已分析歌曲数。
- 展示作词任务数。
- 展示创作任务数。
- 展示 Agent 状态。
- 展示今日和近 7 日 API 执行次数、外部调用次数及文本模型 Token 用量。
- 按供应商展示使用模型、今日用量、近 7 日用量和余额状态。
- 每个分析、作词和音乐任务展示实际使用的供应商、模型、请求方法、接口地址、供应商请求编号和用量。
- 文本模型分别记录输入、输出、缓存和总 Token；音乐平台按其真实计费单位记录积分、次数或音频时长，不把非 Token 用量伪装成 Token。
- 供应商提供官方余额接口时自动查询；未提供时明确显示“控制台查询”，不估算或伪造余额。
- 账户余额只向超级管理员展示，普通成员只看到权限提示。

验收标准：

- 用户登录后能看到清晰工作台。
- 可以查看榜单、分析、歌词、创作任务。
- 新产生的外部 AI 任务能在任务详情和看板中看到真实接口与真实返回用量。
- 历史任务没有用量记录时显示“尚未启用记录”，不能补造 Token。
- 余额查询失败不影响业务任务运行，并显示最后查询状态或人工查询入口。

### FR-008 AI 接口管理

功能要求：

- 超级管理员可以查看、新建、编辑、测试和删除文本模型连接配置。
- 系统预置本地规则、智谱 BigModel、DeepSeek、阿里百炼 Qwen、MiniMax 和通用 OpenAI-compatible 模板；模型名和地址允许按供应商实际账号修改。
- API Key 只以加密密文保存，接口和页面只返回是否已配置及末尾提示，不返回明文。
- 配置必须通过连接测试后才能设为当前使用项；连接测试的接口、耗时和 Token 同样进入统一用量账本。
- 切换当前配置不重启前后端；切换只影响之后开始的调用，已经开始的任务继续使用启动时固定的配置。
- 没有数据库活动配置时继续使用服务器环境变量后备配置，支持将后备配置一次性导入数据库。
- 同一时刻最多存在一个活动文本模型配置，并保留可随时启用的本地规则故障降级配置。

验收标准：

- 超级管理员完成“新建 -> 测试 -> 启用”后，新任务立即使用对应供应商和模型。
- 运行中从本地规则切换到外部模型再切回，不需要重启服务。
- 普通成员不能访问接口配置管理 API 和页面。
- 数据库、API 响应、日志、页面和 Git 仓库均不出现 API Key 明文。
- 活动配置不能被直接编辑或删除，避免无测试修改立即影响业务任务。

### FR-009 演示前端

功能要求：

- 提供一个可给客户演示的前端 demo。
- 视觉风格采用高级、精致、音乐工具感。
- 功能展示包括：榜单、趋势分析、作词、音乐创作、Agent 和后续增强设想。

验收标准：

- 打开 demo 后客户能看懂平台流程。
- 页面能配置并体现“一次点击、按依赖自动执行多个 Agent”的模式。
- 页面能体现“后续工作链/总调度 Agent”是增强功能。

### FR-010 线性自动工作流

功能要求：

- 用户可以选择榜单采集、内容分析和歌词创作步骤并保存流程模板。
- 步骤按固定数据依赖排序，不允许产生无法传参的组合。
- 上一步产出必须真实作为下一步参数，不只按顺序触发独立任务。
- 流程、步骤、子任务和产出都保存状态与编号。
- 流程在后台运行，页面轮询状态；失败时停在当前步骤。

参数关系：

```text
榜单快照和歌曲 entry_ids -> 内容分析
分析 report_id 和创作方向 -> 歌词创作
歌词 version_id 和统一创作方案 -> 音乐生成（P7 接入）
```

验收标准：

- 保存流程后可以一键运行，无需逐页手动点击。
- 运行详情能查看每一步的等待、运行、完成或失败状态。
- 完成步骤能跳转到对应榜单、分析报告或歌词产出。
- 上游失败时下游不执行，页面显示稳定错误码。
- 没有对应 Agent 权限的用户不能创建或运行该步骤。

## 5. 后续增强功能

以下功能不纳入首期，可作为后续加费项目。

### 5.1 高级 Agent 工作链

当前已支持既有步骤的线性组合；后续可以扩展为拖拽式任意工作链。

示例：

```text
酷狗榜单采集 -> 爆款趋势分析 -> 作词 Agent -> 音乐创作 Agent
```

价值：

- 客户能搭建带分支、并行或自定义参数映射的复杂流程。
- 不同 Agent 可以有不同风格和偏好。
- 平台从固定工具升级成可配置工作台。

### 5.2 增强记忆系统

首期作词 Agent 不做独立长期记忆。

后续可升级为更完整记忆系统：

- 长期记录客户创作偏好。
- 根据历史反馈调整作词风格。
- 为不同客户建立不同创作档案。
- 支持多个作词 Agent 拥有不同记忆。

### 5.3 总调度 Agent

用户只描述目标，总调度 Agent 自动拆任务。

示例：

```text
用户说：参考最近热歌，做一首适合短视频传播的治愈系女声歌。
```

系统自动：

```text
采集榜单 -> 分析趋势 -> 生成歌词 -> 创建音乐生成任务 -> 保存结果
```

这是高级功能，不放入首期。
