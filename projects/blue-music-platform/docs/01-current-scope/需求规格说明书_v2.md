# 蓝乐 AI 音乐创作平台 —— 需求规格说明书（精简版）v3.1

## 0. 项目一句话

蓝乐 AI 音乐创作平台是一套面向音乐创作者的 AI 工作台。

它先采集酷狗热歌榜数据，再分析热门歌曲特点，然后由作词 Agent 辅助写歌词，最后通过官方音乐生成 API 生成音乐。

核心链路：

```text
酷狗榜单采集 -> 榜单趋势分析 -> AI 作词 -> 音乐生成 -> 结果展示
```

## 1. 当前交付边界

首期目标是做一个固定、清楚、可演示、可验收的版本。

首期先做：

```text
用户点击一次，系统按固定顺序自动执行多个 Agent
```

暂不做：

```text
用户自定义工作链
总调度 AI 自动理解目标和拆任务
多个增强 Agent 自动协作
复杂长期记忆系统
```

这些放入后续增强项目，等基础项目跑通后再单独报价。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 超级管理员 | 创建和停用账号、分配 Agent 权限、查看团队数据和任务 |
| 团队成员 | 使用被授权的榜单、分析、作词和创作功能 |

## 3. Agent 设计

首期包含 4 个核心 Agent，并由固定工作流统一调度。

| Agent | 作用 | 首期定位 |
|---|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜数据 | 自动采集机器人 |
| 音乐分析 Agent | 根据榜单元数据分析风格、情绪和趋势 | AI 分析助手 |
| 作词 Agent | 根据主题、风格和分析结果生成歌词 | 固定配置的 AI 作词助手 |
| 音乐创作 Agent | 整理素材并调用官方音乐生成 API | 自动提交和结果回收机器人 |

说明：

- “Agent”不一定都代表复杂 AI。
- 有些 Agent 是自动化机器人，例如爬虫 Agent、音乐创作 Agent。
- Agent 之间不直接互相调用，由工作流调度器控制固定执行顺序。
- 作词 Agent 需要 AI 介入；首期使用一个固定配置，独立记忆放入后续增强范围。

## 4. 功能需求

### FR-001 用户登录

原文要求：用户可以登录，系统使用账号密码和令牌保护接口。

精简后要求：

- 支持超级管理员登录。
- 支持内部团队成员登录。
- 不开放用户自助注册，由超级管理员创建和停用账号。
- 密码加密存储。
- 登录后才能使用后台功能。

验收标准：

- 能成功登录后台。
- 错误密码不能登录。
- 登录后能看到工作台。

### FR-002 权限管理

当前要求：系统支持超级管理员和内部团队成员两类角色。

精简后要求：

- 超级管理员可以管理全部功能、账号和 Agent 权限。
- 团队成员只能运行被允许的 Agent。
- 榜单、分析、歌词和创作结果默认在团队内共享。
- 后端接口要做权限校验。

验收标准：

- 团队成员不能访问超级管理员功能。
- 无权限访问时返回错误提示。

### FR-003 酷狗榜单采集

原文中曾出现 QQ 音乐和酷狗音乐不一致。现已确认：

```text
首期先爬酷狗热歌榜。
```

功能要求：

- 支持手动触发酷狗热歌榜采集。
- 采集歌曲名、歌手、排名、时长、来源链接和采集时间；热度、封面仅在公开榜单源提供时保存，缺失时不伪造。
- 数据入库前去重。
- 失败时记录错误信息。
- 按自然日保存榜单快照，默认保留最近 30 天。
- 同一天重复采集时更新当天快照，页面展示最近 15 次运行记录。

验收标准：

- 后台点击按钮后能创建采集任务。
- 采集结果能在列表中展示。
- 同一榜单同一歌曲不重复入库。
- 连续采集后能够按日期查看榜单，并支持默认 7 天趋势分析。

### FR-004 音乐分析

功能要求：

- 对采集到的榜单元数据做基础趋势分析。
- 输出风格、情绪、主题、场景、速度档位、人声、乐器、结构和 Hook 等创作方向。
- 首期不下载酷狗音频，也不承诺基于音频计算真实 BPM、调性和编曲结构。
- 分析结果保存到系统中。

验收标准：

- 用户能选择歌曲并执行分析。
- 分析结果能在后台查看。
- 分析结果能作为作词和创作参考。

### FR-005 作词 Agent

这是本次新增的核心功能。

功能定位：

```text
帮助用户用 AI 写歌词。
```

用户输入：

- 歌曲主题。
- 音乐风格。
- 情绪氛围。
- 目标场景，例如短视频、表白、治愈、夜晚、国风等。
- 想要的人声方向。
- 可选参考歌词或关键词。

功能要求：

- 根据用户输入生成歌词。
- 支持重新生成。
- 保存本次生成结果和错误信息。
- 生成结果整理为统一创作方案；P7 完成后由固定工作流送入音乐创作 Agent。

验收标准：

- 用户能输入主题并生成歌词。
- 用户能保存满意歌词。
- 用户能把歌词带入创作任务。

### FR-006 音乐创作 Agent

功能定位：

```text
把创作素材整理成官方音乐生成 API 可执行的任务。
```

注意：

首期不自研音乐生成模型，音乐生成能力由正式供应商的官方 API 提供。

音乐创作 Agent 负责：

- 整理歌曲标题。
- 整理歌词。
- 整理音乐风格。
- 整理情绪氛围。
- 整理节奏速度。
- 整理人声要求。
- 提交给已配置的音乐生成供应商。
- 跟踪任务状态。
- 保存生成结果。

验收标准：

- 用户能通过固定工作流创建音乐创作任务。
- 系统能展示任务状态：排队中、执行中、完成、失败。
- 音频结果能下载到客户自己的存储并在平台查看。

### FR-007 Dashboard 工作台

功能要求：

- 展示今日采集歌曲数。
- 展示已分析歌曲数。
- 展示作词任务数。
- 展示创作任务数。
- 展示 Agent 状态。

验收标准：

- 用户登录后能看到清晰工作台。
- 可以查看榜单、分析、歌词、创作任务。

### FR-008 演示前端

功能要求：

- 提供一个可给客户演示的前端 demo。
- 视觉风格采用高级、精致、音乐工具感。
- 功能展示包括：榜单、趋势分析、作词、音乐创作、Agent 和后续增强设想。

验收标准：

- 打开 demo 后客户能看懂平台流程。
- 页面能体现“一次点击、固定流程自动执行多个 Agent”的首期模式。
- 页面能体现“后续工作链/总调度 Agent”是增强功能。

## 5. 后续增强功能

以下功能不纳入首期，可作为后续加费项目。

### 5.1 自定义 Agent 工作链

用户可以自己选择多个 Agent 组成流程。

示例：

```text
酷狗榜单采集 -> 爆款趋势分析 -> 作词 Agent -> 音乐创作 Agent
```

价值：

- 客户能自己搭流程。
- 不同 Agent 可以有不同风格和偏好。
- 平台从固定工具升级成可配置工作台。

### 5.2 增强记忆系统

首期作词 Agent 不做独立长期记忆。

后续可升级为更完整记忆系统：

- 长期记录客户创作偏好。
- 根据历史反馈调整作词风格。
- 为不同客户建立不同创作档案。
- 支持多个作词 Agent 拥有不同记忆。

### 5.3 总调度 Agent

用户只描述目标，总调度 Agent 自动拆任务。

示例：

```text
用户说：参考最近热歌，做一首适合短视频传播的治愈系女声歌。
```

系统自动：

```text
采集榜单 -> 分析趋势 -> 生成歌词 -> 创建音乐生成任务 -> 保存结果
```

这是高级功能，不放入首期。

| 层级 | 技术 |

| 后端 | FastAPI |
| 前端 | React + TypeScript |
| 数据库 | PostgreSQL |

| 外部创作工具 | 官方音乐生成 API（Mureka / MiniMax 候选） |

## 8. 版本范围

### 基础版

基础版目标是跑通完整闭环：

```text
登录 -> 一键启动 -> 酷狗榜单 -> 趋势分析 -> AI 作词 -> 音乐生成 -> Dashboard 展示
```

### 完整版

完整版在基础版上增加：

- 多数据源爬取。
- 定时任务。
- 更丰富图表。
- 在线试听。
- 创作版本管理。
- 操作日志。
- 数据备份。

### 后续增强

后续增强单独评估：

- 自定义工作链。
- 多个作词 Agent。
- 多个创作 Agent。
- Agent 长期记忆。
- 总调度 Agent。
