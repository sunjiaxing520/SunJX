# 蓝乐 AI 音乐创作平台项目精炼说明

## 一句话

蓝乐是一个 AI 音乐创作工作台。

它不是直接自研一个音乐生成模型，而是把榜单数据、AI 分析、AI 作词和 Suno 创作串成一个可操作的平台。

## 当前主线

```text
酷狗榜单采集 -> 音乐分析 -> AI 作词 -> Suno 创作 -> 结果展示
```

## 首期核心价值

- 先看市场：采集酷狗热歌榜。
- 再做分析：分析热门歌曲的风格、情绪、关键词。
- 然后写词：作词 Agent 根据主题和用户偏好写歌词。
- 最后创作：Suno 创作 Agent 把素材整理后提交给 Suno。
- 统一展示：平台保存任务、结果和历史记录。

## 首期 Agent

| Agent | 作用 |
|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜 |
| 音乐分析 Agent | 分析歌曲风格、情绪、关键词 |
| 作词 Agent | AI 写歌词，并记住基础偏好 |
| Suno 创作 Agent | 整理素材，提交 Suno，回收结果 |

## 作词 Agent 为什么重要

客户想用 AI 写歌词。

歌词不是一次性表单，它会受到用户偏好影响，所以作词 Agent 需要基础记忆：

```text
常用主题
喜欢的歌词风格
常用表达方式
不想要的词
历史歌词记录
```

首期只做基础记忆，不做复杂长期记忆系统。

## 当前交付边界

首期做：

```text
人操作多个固定 Agent
```

不做：

```text
用户自定义工作链
总调度 Agent
多个作词 Agent 不同记忆
复杂长期记忆系统
多个 Agent 自动协作
```

这些都作为后续加费项目。

## 给客户的讲法

可以这样讲：

```text
我们先做一个清晰可控的版本：系统采集酷狗热歌榜，分析热门歌曲特征，再由作词 Agent 写歌词，最后由 Suno 创作 Agent 提交到 Suno 生成音乐。后续如果效果好，可以升级为工作链和总调度 Agent，让客户自己组合多个 Agent 或让系统自动调度。
```
