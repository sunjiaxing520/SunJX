# 蓝乐 AI 音乐创作平台项目精炼说明

## 一句话

蓝乐是一个 AI 音乐创作工作台。

它不是直接自研一个音乐生成模型，而是把榜单数据、AI 分析、AI 作词和正式音乐生成 API 串成一个可操作的平台。

## 当前主线

```text
可选酷狗榜单采集 -> 音乐分析 -> AI 作词 -> 正式音乐生成 API -> 结果展示
```

## 首期核心价值

- 先看市场：采集酷狗热歌榜。
- 再做分析：分析热门歌曲的风格、情绪、关键词。
- 然后写词：作词 Agent 根据主题和用户偏好写歌词。
- 最后创作：音乐创作 Agent 把统一创作方案提交给正式供应商。
- 统一展示：平台保存任务、结果和历史记录。

## 首期 Agent

| Agent | 作用 |
|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜 |
| 音乐分析 Agent | 分析歌曲风格、情绪、关键词 |
| 作词 Agent | AI 写歌词，并记住基础偏好 |
| 音乐创作 Agent | 整理素材，提交 Suno 官方 API，回收、试听、下载和续写结果；集成基础已完成，真实调用待官方凭证 |

## 作词 Agent 为什么重要

客户想用 AI 写歌词。

歌词不是一次性表单，它会受到用户偏好影响，所以作词 Agent 需要基础记忆：

```text
常用主题
喜欢的歌词风格
常用表达方式
不想要的词
历史歌词记录
```

首期只做基础记忆，不做复杂长期记忆系统。

## 当前交付边界

当前已做：

```text
用户选择既有 Agent 组成线性流程，一次启动后自动传递每一步产出
```

不做：

```text
拖拽式任意工作链、分支、并行和循环
总调度 Agent
多个作词 Agent 不同记忆
复杂长期记忆系统
增强 Agent 自动选择和协作
```

这些都作为后续加费项目。

## 给客户的讲法

可以这样讲：

```text
我们先做一个清晰可控的版本：用户可以选择榜单采集、内容分析和作词步骤，系统自动把上一步结果传给下一步。正式音乐接口接入后会继续把歌词和创作方案传给音乐 Agent。后续还可以升级条件分支、并行流程和总调度 Agent。
```
