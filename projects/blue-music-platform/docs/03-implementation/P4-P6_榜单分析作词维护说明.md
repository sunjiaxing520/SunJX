# P4-P6 榜单、分析和作词维护说明

## 当前交付

```text
酷狗 TOP500 实时采集
-> 按自然日保存快照
-> 默认读取连续 7 天排名变化
-> 输出 3 个结构化创作方向
-> 生成并保存标准段落歌词
-> 导出音乐平台通用创作方案
```

数据库默认保留最近 30 天榜单快照。同一天重复采集会替换当天条目，并把 `collected_at` 更新为最后一次成功采集时间；采集任务仍单独记录。页面最多展示最近 15 个快照结果和 15 次运行记录。采集结果默认折叠，每个快照只显示日期、榜单、歌曲数和采集时间；点击某个快照后才请求并在结果块内展开该日排行榜详情，一次只展开一个快照。

## 任务记录主动删除

采集运行记录、分析记录和作词记录都支持勾选后批量删除，也支持每条记录单独删除。三类批量接口都接收 `task_ids`，一次最多 100 条并自动去重；批量请求包含不存在或仍在运行的任务时整批拒绝，不会只删一部分。

- 删除采集运行记录只删除 `collection_tasks`，不删除 `ranking_snapshots` 和歌曲条目；自动流程保留快照编号。
- 删除分析记录会删除任务、报告和分析收藏；引用该报告的既有歌词继续保留，但解除来源报告和方向编号。
- 删除作词记录会删除任务、全部歌词版本和歌词收藏。
- 分析和作词的 `api_usage_records` 继续保留，用于 Token、费用和故障审计。
- 自动流程里的失效任务和产出编号会清空；采集步骤仍有快照时显示“运行记录已删除，快照保留”，分析或作词产出删除后显示“产出已删除”。

## 团队收藏夹

`favorite_items` 保存团队共享的收藏关系，不复制分析正文或歌词正文：

- `analysis + target_id` 指向一份具体分析报告。
- `lyrics + target_id` 指向一个具体歌词版本。
- 同一目标只有一条团队收藏，重复创建返回已有记录。
- 备注最多 2000 字，空白备注会标准化为 `null`。
- 原始产物生成时间与收藏时间分别展示；移出收藏夹只删除收藏关系。
- 来源账号删除后 `created_by_id` 自动置空，收藏和原始产物继续保留。
- 收藏来源不受最近 15 条历史列表限制，页面会按任务编号请求详情；后续自动数据清理必须跳过仍被收藏的报告和歌词版本。用户主动确认删除分析或作词任务时，对应收藏会同步删除。

收藏页一次读取最多 100 个精选条目，按收藏时间倒序并按类别筛选，页面每页展示 8 条。当前属于低并发内部团队功能；条目量显著增长后再改为服务端分页。

## 文本 AI 配置

系统按以下优先级选择文本模型：

1. 读取数据库中唯一的活动 `ai_provider_configs` 配置。
2. 没有数据库活动配置时，使用 `.env` 中的 `AI_*` 后备配置。
3. 后备配置为 `local` 时使用免费、可复现的本地规则基线。

超级管理员在“AI 接口”页面新增、测试并启用配置。切换会在下一次分析或作词调用开始时立即生效，不需要重启后端；已经开始的任务持有启动时解析出的 Provider，不会在执行中途换接口。

系统预置本地规则、智谱 BigModel、DeepSeek、阿里百炼 Qwen、MiniMax 和通用 OpenAI-compatible 模板。模板提供官方 Base URL、默认模型和协议差异，实际模型名仍可编辑。

仓库后备配置：

```env
AI_PROVIDER=local
```

`local` 用于未配置环境、自动化测试和外部平台故障降级，不应对客户宣传为正式大模型效果。

当前本机已把 `.env` 中的智谱 `glm-4.7-flash` 一次性导入数据库并启用，同时保留环境变量作为后备。API Key 使用 Fernet 认证加密保存，API 和页面只返回末尾提示。

选定正式文本模型后配置：

```env
AI_PROVIDER=openai_compatible
AI_BASE_URL=https://供应商的兼容接口/v1
AI_API_KEY=只保存在服务器环境变量中
AI_MODEL=供应商模型名
AI_CONFIG_ENCRYPTION_KEY=长期稳定且只存在于服务器的加密主密钥
AI_REQUEST_TIMEOUT_SECONDS=180
AI_MAX_RETRIES=2
AI_ANALYSIS_MAX_OUTPUT_TOKENS=2500
AI_LYRICS_MAX_OUTPUT_TOKENS=3500
```

正式模型返回值仍要经过 Pydantic 结构校验。缺字段、错误类型、超时或无效 JSON 都会令任务失败并保存错误码。
智谱 GLM-4.7/5 系列默认会思考；当前结构化分析和作词不需要深度推理，适配器仅对该系列发送 `thinking.type=disabled`，并使用上述输出上限控制时延和用量。其他 OpenAI-compatible 供应商不会收到这个智谱专用参数。

`AI_CONFIG_ENCRYPTION_KEY` 未配置时暂时回退到 `SECRET_KEY`。生产部署应显式设置独立值并长期保留；修改或丢失该值会导致已经保存的 API Key 无法解密，需要重新录入。活动配置禁止直接编辑和删除，必须先切换到其他已测试配置。

## API 用量记录

分析和作词每次执行都会新增一条 `api_usage_records` 记录：

- 记录任务类型和任务编号、供应商、模型、请求方法、接口地址及供应商请求编号。
- 智谱等 OpenAI-compatible 响应中的 `usage.prompt_tokens`、`usage.completion_tokens`、缓存 Token 和总 Token 会按原值保存。
- 本地规则执行也会留下记录，但明确标记为非外部调用且 Token 为 0。
- 失败调用保存接口、重试次数、耗时和错误信息；供应商未返回用量时保持 0，不做字符数估算。
- 旧任务不会补造 Token，因此页面会显示“历史任务未记录接口”。
- Dashboard 按应用时区统计今日和近 7 日用量。当前智谱适配器没有可用的官方余额查询接口，超级管理员通过供应商控制台查看；后续 Mureka 可接入其官方 `/v1/account/billing`。

## 分析边界

- 只使用榜单元数据和排名变化，不下载酷狗音频。
- 曲风、情绪、乐器和人声是创作建议，不是对原曲的事实检测。
- 不输出准确 BPM、调性、和弦或要求复刻具体歌曲、歌手。
- 有效数据不足 7 天时仍可分析，但报告会显示较低或中等可信度。
- 酷狗公开榜单未提供的热度和封面保持为空，不制造假数据。

## 云端注意事项

- `APP_TIMEZONE=Asia/Shanghai` 统一快照日期和 Dashboard 的“今日”边界。
- 生产环境执行 `alembic upgrade head` 后再启动新版后端。
- `favorite_items` 使用 Alembic 迁移创建，云端升级前必须先备份 PostgreSQL，再执行迁移。
- `api_usage_records` 从功能上线后开始记录，数据库迁移不会伪造旧任务用量。
- 当前是低并发内部工具，采集、分析和作词在 HTTP 请求内完成；若正式模型响应经常超过一分钟，再将任务执行迁移到 Redis 队列，不提前增加维护成本。
- Docker 和云端部署必须给所有后端实例提供相同且稳定的 `AI_CONFIG_ENCRYPTION_KEY`；滚动部署不影响数据库活动配置和热切换。
- 酷狗页面结构变化时先运行解析器测试，再根据 `CRAWLER_SOURCE_UNAVAILABLE` 任务编号检索日志。
