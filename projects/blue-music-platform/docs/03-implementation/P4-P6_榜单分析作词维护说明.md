# P4-P6 榜单、分析和作词维护说明

## 当前交付

```text
酷狗 TOP500 实时采集
-> 按自然日保存快照
-> 默认读取连续 7 天排名变化
-> 输出 3 个结构化创作方向
-> 生成并保存标准段落歌词
-> 导出音乐平台通用创作方案
```

数据库默认保留最近 30 天榜单快照。同一天重复采集会替换当天条目，采集任务仍单独记录；页面最多展示最近 15 次运行记录。

## 主要接口

```text
POST /api/v1/rankings/collections
GET  /api/v1/rankings/collections
GET  /api/v1/rankings/snapshots
GET  /api/v1/rankings/entries

POST /api/v1/analysis/tasks
GET  /api/v1/analysis/tasks
GET  /api/v1/analysis/tasks/{task_id}

POST /api/v1/lyrics/tasks
POST /api/v1/lyrics/tasks/{task_id}/regenerate
PUT  /api/v1/lyrics/versions/{version_id}/save
GET  /api/v1/lyrics/versions/{version_id}/creation-brief
```

实时采集失败会返回 `502` 并留下失败任务。固定样例只能通过采集按钮旁的维护菜单明确触发，不会在实时失败时自动冒充成功。

## 文本 AI 配置

开发环境默认：

```env
AI_PROVIDER=local
```

`local` 是免费、可复现的规则基线，用于开发联调和自动化测试，不应对客户宣传为正式大模型效果。

选定正式文本模型后配置：

```env
AI_PROVIDER=openai_compatible
AI_BASE_URL=https://供应商的兼容接口/v1
AI_API_KEY=只保存在服务器环境变量中
AI_MODEL=供应商模型名
AI_REQUEST_TIMEOUT_SECONDS=60
AI_MAX_RETRIES=2
```

正式模型返回值仍要经过 Pydantic 结构校验。缺字段、错误类型、超时或无效 JSON 都会令任务失败并保存错误码。

## 分析边界

- 只使用榜单元数据和排名变化，不下载酷狗音频。
- 曲风、情绪、乐器和人声是创作建议，不是对原曲的事实检测。
- 不输出准确 BPM、调性、和弦或要求复刻具体歌曲、歌手。
- 有效数据不足 7 天时仍可分析，但报告会显示较低或中等可信度。
- 酷狗公开榜单未提供的热度和封面保持为空，不制造假数据。

## 云端注意事项

- `APP_TIMEZONE=Asia/Shanghai` 统一快照日期和 Dashboard 的“今日”边界。
- 生产环境执行 `alembic upgrade head` 后再启动新版后端。
- 当前是低并发内部工具，采集、分析和作词在 HTTP 请求内完成；若正式模型响应经常超过一分钟，再将任务执行迁移到 Redis 队列，不提前增加维护成本。
- 酷狗页面结构变化时先运行解析器测试，再根据 `CRAWLER_SOURCE_UNAVAILABLE` 任务编号检索日志。
