## 4. 未完全通过项

### Docker Compose 后端服务未通过

命令：

```powershell
cd D:\SunJX\projects\blue-music-platform
docker compose up -d backend
```

结果：失败。

失败原因：

```text
Docker 构建 backend 镜像时需要拉取 python:3.11-slim。
当前环境无法连接 Docker Hub 授权接口，导致镜像元数据拉取失败。
```

关键错误：

```text
failed to fetch anonymous token
Get "https://auth.docker.io/token?...": connectex
```

判断：

```text
这是 Docker Hub 网络访问问题，不是后端代码测试失败。
但它影响“Docker Compose 可以启动 backend”这一项验收标准。
```

## 5. 已知问题

- `docker-compose.yml` 中的 `version` 字段会被新版 Docker Compose 提示为 obsolete，目前不影响 PostgreSQL 和 Redis 运行。
- Docker Desktop 依赖 Windows `VirtualMachinePlatform` 和 Hypervisor；切换到游戏模式后 Docker 会不可用。
- `docker compose up -d backend` 依赖 Docker Hub 拉取 `python:3.11-slim`，当前网络环境下未通过。
