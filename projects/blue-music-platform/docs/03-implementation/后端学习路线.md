# 后端学习路线

本文档记录蓝乐项目后端开发时的学习顺序。

## 第一阶段：先吃透请求流程

先从最简单的健康检查接口开始：

```text
浏览器或前端请求
-> FastAPI 应用入口 main.py
-> API 总路由 api/v1/router.py
-> 具体接口 api/v1/routes/health.py
-> 响应结构 schemas/health.py
-> 返回 JSON
```

这一阶段先理解：

- `main.py` 为什么是后端入口。
- `router.py` 为什么负责汇总接口。
- `routes/` 为什么放具体接口。
- `schemas/` 为什么定义请求和响应结构。
- `core/config.py` 为什么集中读取环境变量。

## 第二阶段：理解数据库

下一步再接数据库：

```text
接口
-> service 业务逻辑
-> SQLAlchemy Session
-> PostgreSQL
-> 返回数据
```

这一阶段会重点学习：

- 数据库连接字符串是什么。
- ORM 是什么。
- Model 和数据库表是什么关系。
- 为什么接口里不要直接堆数据库操作。

## 第三阶段：第一个真实业务闭环

第一个真实业务模块建议做登录：

```text
用户输入邮箱和密码
-> POST /api/v1/auth/login
-> 后端校验用户和密码
-> 生成 JWT
-> 前端保存 token
-> 后续请求携带 Authorization
```

登录模块能把前端请求、后端接口、数据库、密码加密、权限保护串起来。
