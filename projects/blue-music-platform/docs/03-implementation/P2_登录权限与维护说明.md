# P2 登录权限与维护说明

状态：已完成，自动化测试与真实 PostgreSQL 联调通过。

对应需求：FR-001、FR-002。

## 1. 实现范围

- 超级管理员和内部团队成员登录。
- 不开放自助注册。
- Argon2 密码哈希，数据库不保存明文密码。
- JWT Access Token，默认有效期 120 分钟。
- 每次受保护请求重新查询用户状态和权限。
- 超级管理员创建、停用成员和重置成员密码。
- 超级管理员分配爬虫、分析、作词和音乐创作 Agent 权限。
- 普通成员不能访问管理员接口。
- 安全、幂等的初始超级管理员命令。
- 配置、PostgreSQL、Redis 和 Alembic 一键诊断命令。
- 登录、管理员操作和权限变更审计日志。

首期不实现公开注册、邮箱找回密码、Refresh Token、OAuth、MFA 和复杂 RBAC。

## 2. 数据模型

### users

```text
id
username
password_hash
token_version        用于使旧Token立即失效
role                 super_admin 或 member
is_active
created_at
updated_at
```

### user_agent_permissions

```text
user_id
agent                crawler、analysis、lyrics 或 music
created_at
```

`user_id + agent` 是联合主键，同一个成员不会重复拥有同一权限。删除用户时，权限记录通过外键级联删除。

超级管理员不需要权限记录，后端默认允许超级管理员使用全部 Agent。

## 3. 接口

| 方法 | 地址 | 权限 | 功能 |
|---|---|---|---|
| `POST` | `/api/v1/auth/login` | 公开 | 账号密码登录 |
| `GET` | `/api/v1/auth/me` | 已登录 | 获取当前用户 |
| `GET` | `/api/v1/users` | 超级管理员 | 查看账号列表 |
| `POST` | `/api/v1/users` | 超级管理员 | 创建成员账号 |
| `PATCH` | `/api/v1/users/{id}/status` | 超级管理员 | 启用或停用账号 |
| `PUT` | `/api/v1/users/{id}/password` | 超级管理员 | 重置成员密码 |
| `PUT` | `/api/v1/users/{id}/agent-permissions` | 超级管理员 | 替换 Agent 权限集合 |

系统没有注册接口，普通用户无法自行创建账号或提升角色。

## 4. 登录流程

```text
POST /api/v1/auth/login
-> LoginRequest 校验账号和密码格式
-> services/auth.py 查询用户
-> Argon2 验证密码哈希
-> 检查 is_active
-> core/security.py 生成 JWT
-> 返回 access_token
```

JWT 只保存用户编号、Token 类型、签发时间、过期时间和随机编号，不保存密码和权限。

受保护请求：

```text
Authorization: Bearer <access_token>
-> HTTPBearer 提取 Token
-> 校验签名和过期时间
-> 根据 sub 查询 users 表
-> 检查 is_active
-> 检查超级管理员或 Agent 权限
-> 执行接口
```

因为每次请求都查询数据库，停用账号或修改权限会立即生效，不需要等待旧 Token 过期。重置密码或停用账号时还会递增 `token_version`，已经签发的旧 Token 即使账号重新启用也不能恢复使用。

## 5. 初始化超级管理员

本地配置从项目根目录 `.env` 读取；云服务器直接使用容器或系统环境变量，并且云端环境变量优先于 `.env`。

先执行迁移：

```powershell
cd D:\SunJX\projects\blue-music-platform\backend
D:\DevTools\Venvs\blue-music-backend\Scripts\python.exe -m alembic upgrade head
```

交互式创建，密码输入不会显示在终端：

```powershell
D:\DevTools\Venvs\blue-music-backend\Scripts\python.exe -m app.cli.create_superadmin --username admin
```

自动部署可以临时使用：

```text
INITIAL_SUPERADMIN_USERNAME
INITIAL_SUPERADMIN_PASSWORD
```

初始化完成后应立即从部署环境删除 `INITIAL_SUPERADMIN_PASSWORD`。

命令是幂等的：同名超级管理员已存在时不会重复创建，也不会自动重置密码；同名成员存在时拒绝将其提升为超级管理员。

## 7. 一键诊断

本地或云服务器进入 backend 目录后运行：

```powershell
D:\DevTools\Venvs\blue-music-backend\Scripts\python.exe -m app.cli.diagnose
```

云端容器内示例：

```bash
docker compose exec -T backend python -m app.cli.diagnose
```

诊断报告检查：

- SECRET_KEY 是否适合当前环境。
- JWT 算法和过期时间。
- PostgreSQL 是否能执行 SQL。
- 数据库地址是否已脱敏。
- Redis 是否返回成功。
- 数据库迁移版本是否等于代码最新版本。
- 日志等级、格式和输出目标。

报告状态：

```text
ok       全部正常
warning  可以开发，但存在非生产配置，例如开发默认密钥
error    数据库、Redis、迁移或生产安全配置有问题
```

诊断命令返回 `error` 时退出码为 1，方便云端部署脚本阻止错误版本继续发布。

## 8. 维护定位

```text

```

需要维护时，优先提供：

```text

error.code

```

错误码和查日志方法见 `错误定位与错误码手册.md`。

## 9. 生产安全

生产环境启动时会拒绝默认或少于 32 字符的 `SECRET_KEY`。

生成随机密钥示例：

```powershell
D:\DevTools\Venvs\blue-music-backend\Scripts\python.exe -c "import secrets; print(secrets.token_urlsafe(48))"
```

生产部署要求：

```text
SECRET_KEY 使用服务器环境变量或密钥服务
ENVIRONMENT=production
LOG_FORMAT=json
LOG_TO_FILE=false
先执行 alembic upgrade head
再启动或更新 backend
最后运行 app.cli.diagnose
```

## 11. 验收覆盖

自动化测试覆盖：

- 密码以 Argon2 哈希保存。
- 正确密码可以登录和获取当前用户。
- 错误密码和不存在账号返回相同错误。
- 缺失、伪造和过期 Token 返回 401。
- 普通成员访问管理员接口返回 403。
- 管理员可以创建成员、重置密码、停用账号和分配 Agent 权限。
- 没有 Agent 权限时接口返回 403，授权后立即通过。
- 停用账号后旧 Token 立即失效。
- 重置密码后旧 Token 返回 `AUTH_TOKEN_REVOKED`。
- 重复账号和管理员停用自己被阻止。
- 初始化命令幂等且不保存明文密码。
- 生产环境拒绝不安全的 SECRET_KEY。
- 诊断输出不暴露数据库密码。
