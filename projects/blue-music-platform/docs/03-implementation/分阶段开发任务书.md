# 蓝乐 AI 音乐创作平台分阶段开发任务书

版本：v1.0  
制定日期：2026-06-12  
依据：需求规格说明书 v3.0、项目决策记录

## 1. 项目目标

首期交付一个固定、清楚、可操作、可演示、可验收的 AI 音乐创作工作台。

完整业务闭环：

```text
登录
-> 酷狗榜单采集
-> 音乐分析
-> AI 作词
-> Suno 创作任务
-> Dashboard 和结果展示
```

首期采用“用户手动操作多个固定 Agent”的模式。

首期不包含：

- 用户自定义工作链。
- 总调度 Agent 自动拆分任务。
- 多 Agent 自动协作。
- 复杂长期记忆系统。
- 多音乐平台数据源。
- 自研音乐生成模型。

## 2. 执行原则

1. 每个阶段必须完成代码、测试、文档和讲解后才能验收。
2. 上一阶段未通过验收，不开始依赖它的下一阶段。
3. 每个功能先跑通最小闭环，再增加细节。
4. 外部服务必须通过适配层接入，业务代码不能直接绑定单一供应商。
5. 所有密钥、密码和连接地址通过环境变量提供，不写死在代码中。
6. 每次开发都要说明请求从哪里进入、经过哪些文件、数据如何保存、结果如何返回。

## 5. P1：后端基础架构和数据库

### 开发任务

- 完成 FastAPI 应用工厂、配置、路由、Schema、Service、Model 分层。
- 配置 PostgreSQL、Redis 和 Docker Compose。
- 建立 SQLAlchemy 数据库连接和会话管理。
- 初始化 Alembic 数据库迁移。
- 建立统一响应和异常处理基础。
- 增加应用健康检查和数据库健康检查。
- 为基础代码增加自动化测试。

### 学习目标

- 理解 HTTP 请求和 JSON 响应。
- 理解 FastAPI 应用、路由和依赖注入。
- 理解环境变量、数据库连接字符串、ORM、Session 和迁移。
- 能解释 `api`、`schemas`、`services`、`models`、`core` 的职责。

### 交付物

- 可启动的 FastAPI 服务。
- 可连接的 PostgreSQL 和 Redis。
- Alembic 初始化配置。
- 健康检查接口和基础测试。
- 后端启动与代码流程说明。

### 完成标准

- `GET /api/v1/health` 返回 HTTP 200。
- 数据库健康检查能明确返回连接成功或失败原因。
- Docker Compose 可以启动 PostgreSQL、Redis 和 backend。
- Alembic 可以执行升级和降级操作。
- 测试命令通过，代码中没有提交本地密码或密钥。
- 学习者可以独立讲出一次健康检查请求经过的文件顺序。

## 6. P2：用户登录和权限

对应 SRS：FR-001、FR-002。

### 开发任务

- 设计用户、角色和必要权限数据模型。
- 实现管理员和普通用户账号。
- 实现密码哈希存储和密码校验。
- 实现登录、获取当前用户和 Token 校验接口。
- 使用 JWT 保护后台接口。
- 实现管理员接口权限校验。
- 准备初始化管理员的安全方式。
- 增加认证和越权测试。

### 学习目标

- 理解数据库 Model、Schema 和 Service 的关系。
- 理解密码为什么不能明文保存。
- 理解 JWT 的生成、携带、校验和过期过程。
- 理解 HTTP 401 和 403 的区别。

### 交付物

- 用户和角色数据库迁移。
- 登录接口、当前用户接口、权限依赖。
- 管理员和普通用户测试账号初始化方案。
- Swagger 接口说明和认证流程图。

### 完成标准

- 管理员和普通用户都能使用正确密码登录。
- 错误密码无法登录，数据库中不存在明文密码。
- 未携带有效 Token 时不能访问受保护接口。
- 普通用户访问管理员接口时返回 403。
- 管理员可以访问管理员接口。
- 认证相关自动化测试全部通过。
- 学习者能从前端登录请求讲到后端生成 Token 的完整过程。

## 8. P4：酷狗榜单采集 Agent

对应 SRS：FR-003。

### 开发任务

- 设计榜单歌曲和采集任务数据模型。
- 数据模型保留 `platform` 字段。
- 实现手动创建采集任务的接口和页面。
- 实现酷狗数据源适配器。
- 采集歌曲名、歌手、排名、热度、封面和采集时间。
- 实现数据清洗、去重、任务状态和错误记录。
- 实现歌曲列表、分页和基础筛选。
- 为第三方页面变化准备模拟数据或导入兜底方案。

### 学习目标

- 理解第三方数据源适配层。
- 理解任务状态、幂等和数据库唯一约束。
- 理解同步请求和耗时任务的区别。
- 理解爬虫不稳定性及错误日志的重要性。

### 交付物

- 采集任务表和榜单歌曲表。
- 手动采集接口和操作页面。
- 酷狗适配器、去重逻辑和错误记录。
- 榜单歌曲列表页面。

### 完成标准

- 后台点击按钮可以创建一条采集任务。
- 任务状态至少支持等待、执行中、完成和失败。
- 成功任务可以产生并展示榜单歌曲数据。
- 同一榜单同一首歌曲重复采集时不会重复入库。
- 失败时能看到错误原因，系统不会整体崩溃。
- 适配器测试可以使用固定样本稳定复现解析结果。

## 9. P5：音乐分析 Agent

对应 SRS：FR-004。

### 开发任务

- 设计音乐分析结果和分析任务数据模型。
- 实现选择歌曲并创建分析任务。
- 建立 AI 模型适配层和结构化输出格式。
- 输出风格、情绪、关键词和节奏倾向。
- 实现解析失败、超时、重试和错误记录。
- 保存并展示分析结果。
- 允许作词和创作模块引用分析结果。

### 学习目标

- 理解外部 API 请求、超时和重试。
- 理解 Prompt、结构化输出和数据校验。
- 理解为什么外部 AI 返回值不能直接信任。
- 理解分析结果与原始歌曲数据的关联关系。

### 交付物

- 音乐分析数据模型、接口、Service 和页面。
- AI 模型适配器。
- 分析 Prompt 和结构化响应 Schema。
- 分析任务测试和错误处理。

### 完成标准

- 用户可以选择至少一首歌曲创建分析任务。
- 成功结果包含风格、情绪、关键词和节奏倾向。
- 分析结果被保存，刷新页面后仍可查看。
- 失败任务有明确状态和错误信息。
- 同一结果可以被作词任务或创作任务引用。
- 使用模拟适配器时自动化测试不依赖真实付费 API。

## 10. P6：作词 Agent 和基础记忆

对应 SRS：FR-005。

### 开发任务

- 设计作词任务、歌词版本和用户歌词偏好模型。
- 实现主题、风格、情绪、场景、人声、关键词等输入。
- 支持引用音乐分析结果。
- 实现生成、重新生成、保存歌词。
- 实现重写主歌、副歌及风格调整等局部修改。
- 读取用户常用主题、偏好风格、常用表达和禁用内容。
- 保存历史歌词记录。
- 实现“一键送入 Suno 创作任务”。

### 学习目标

- 理解一次性任务数据和用户偏好数据的区别。
- 理解父任务、生成版本和历史记录的关系。
- 理解基础记忆不是让模型自行永久记忆，而是后端保存并按需注入上下文。
- 理解局部修改如何保留原始歌词和生成新版本。

### 交付物

- 作词任务、歌词版本和偏好数据模型。
- 作词 Agent 接口、Service、适配器和页面。
- 生成、重新生成、局部修改、保存和转创作流程。
- 基础记忆说明和测试。

### 完成标准

- 用户输入主题和必要条件后能获得一份结构清楚的歌词。
- 用户可以重新生成并保留历史版本。
- 用户可以针对主歌或副歌提出局部修改。
- 满意歌词可以保存，刷新页面后仍存在。
- Agent 会读取当前用户的基础偏好，且不同用户数据互不混淆。
- 保存的歌词能直接创建预填充的 Suno 创作任务。

### 学习目标

- 理解轮询、回调、超时和状态机。
- 理解适配层如何降低第三方平台变化的影响。
- 理解为什么不能在前端暴露第三方 API Key。

### 交付物

- 创作任务和结果模型。
- 创作工具接口、Suno 适配器和模拟适配器。
- 创建任务、查询状态和保存结果的接口与页面。
- 外部服务异常处理和测试。

### 完成标准

- 用户可以从空白表单或已保存歌词创建创作任务。
- 任务状态至少支持排队中、执行中、完成和失败。
- 完成任务能够保存并展示结果链接。
- 失败任务能够展示可理解的失败原因。
- API Key 只存在于后端环境配置中。
- 切换模拟适配器和真实适配器不需要改动核心业务流程。
