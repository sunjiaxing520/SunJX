# 蓝乐 AI 音乐创作平台 —— 客户讲解手册

> 使用说明：每一节分为「原文」和「怎么讲」两部分，中间用分隔线隔开。原文是 SRS 和报价方案的客观摘录，怎么讲是面向客户的口语化表达。讲的时候先读「怎么讲」，客户追问细节时再翻「原文」佐证。

## 一、项目概述

### 原文

**产品定位**（来源：报价方案 1.1）

蓝乐 AI 音乐创作平台是一款基于 AI Agent 技术的智能化音乐创作辅助平台。平台通过智能爬虫采集全网音乐榜单数据，结合多维分析 Agent 深入解析音乐特征，最终辅助用户完成高质量音乐创作。核心理念是「数据驱动创作，AI 赋能灵感」。

**技术栈**（来源：报价方案 第二章）

| 层级 | 技术选型 | 版本要求 |
|---|---|---|
| 后端框架 | FastAPI (Python) | 3.10+ |
| 前端框架 | React + TypeScript | 18+ |
| 数据库 | PostgreSQL | 15+ |
| 缓存 | Redis | 7+ |
| 容器化 | Docker + Compose | 24+ |

---

### 怎么讲

**一句话概括**：这个平台就是音乐创作领域的"智能参谋 + 自动助手"——它帮你自动扒全网热歌榜数据、分析歌曲用了什么风格什么和弦、然后操控创作工具帮你出活儿。

**竞品对比（核心卖点）**：市面上现有的音乐 AI 工具，可以理解成"写死的一体机"——打开就是那几个功能，想加新能力？对不起，得换一台。我们做的，是"模块化组装机"——基础版配 3 个智能模块，将来你想加情绪分析、加网易云数据源、加自动备份，插上去就行，不用重做系统。

**技术栈大厂背书**：

- FastAPI——和 Uber、Netflix 同款后端框架，异步高性能，接口响应毫秒级
- React + TypeScript——和 Facebook、Airbnb 同款前端方案，界面流畅不卡顿
- PostgreSQL + Redis——全球排名前 3 的数据库组合，银行级事务处理，数据不会丢
- Docker 容器化——部署就和装 App 一样简单，一台服务器一条命令全部跑起来

## 二、基础版 6K——逐个功能模块

### 2.1 用户管理（注册 / 登录 / 个人信息）

#### 原文

**FR-0001 用户注册**：新用户通过邮箱或用户名注册。用户名 3-20 字符，密码 8-32 字符（含大小写字母+数字）。密码经 bcrypt 哈希存储，注册成功自动分配 user 角色。

**FR-0002 用户登录**：JWT Token 认证，Access Token 2 小时有效，Refresh Token 7 天有效。连续 5 次输错锁定 15 分钟。

**FR-0003 个人信息管理**：可编辑昵称、头像、简介。修改邮箱/密码需验证当前密码。

---

#### 怎么讲

**干什么**：就是平台的"账号系统"——注册、登录、改资料，和用任何一个 App 一样。

**通俗类比**：好比进健身房，先办卡（注册），刷卡进（登录），想换照片改资料随时可以（个人信息），卡丢了挂失重办（改密码需验证）。

**为什么比竞品好**：竞品很多是"单机版"，账号密码存本地，换个电脑就没了。我们是云端账号体系，你在公司登录、回家登录、手机上登录，数据全同步。密码用的是 bcrypt 加密——和银行同一个级别，就算数据库泄露你的密码也解不出来。

---

### 2.2 权限管理 RBAC（管理员 + 普通用户两角色）

#### 原文

**FR-0004 角色定义**：系统预置 admin 和 user 两种角色。admin 全部 API 可访问，user 仅业务操作和 Dashboard 查看。

**FR-0005 权限校验**：API 网关层基于 JWT 角色字段校验。无权限返回 403，过期 Token 返回 401。

**FR-0006 角色管理**：管理员可查看用户列表、修改用户角色，角色变更后当前 Token 自动失效。

---

#### 怎么讲

**干什么**：分两把钥匙——团队 Leader 拿"总钥匙"（admin），能看到所有数据、改别人权限、配系统设置；成员拿"工作钥匙"（user），只能用自己的功能模块。

**通俗类比**：就像公司门禁系统——老板的卡刷得开所有门，员工的卡只能进自己的工区和会议室。有人离职，老板后台一关，他的卡就失效了，不用追着他要回来。

**为什么比竞品好**：竞品很多没有权限分级，要么大家共用一套账号（谁改了设置都不知道），要么根本不分角色。我们的 RBAC 是 API 网关层做权限拦截——不是在页面上"藏"按钮那种假权限，而是从后端接口层面就拦住了，安全性不在一个级别。

---

### 2.3 Agent 热插拔框架（3 槽位，手动配置）

#### 原文

**FR-0007 Agent 注册与发现**：Agent 启动时注册元信息（名称、类型、能力描述、槽位占用）。基础版最多 3 个槽位。

**FR-0008 Agent 配置管理**：管理员手动启用/禁用/配置 Agent，变更需手动保存生效。

**FR-0010 心跳与健康检查**：心跳间隔 10 秒，连续 3 次丢失判定异常。Dashboard 实时显示各 Agent 运行状态。

---

#### 怎么讲

**干什么**：平台的核心骨架。就像一个"USB 集线器"——基础版自带 3 个插口，你可以插爬虫模块、分析模块、创作模块。哪个模块需要升级，拔下来换新的，其他模块照常工作。

**通俗类比**：你家电视柜有 3 个 HDMI 接口。你现在接了游戏机（爬虫）、机顶盒（分析）、蓝光播放器（创作）。将来想加个 Apple TV？8K 版就是多给你 2 个口（共 5 个），而且支持热插拔——插上就用，不用关电视重启。

**为什么比竞品好**：竞品是"焊死的电路板"——想加功能？返厂。我们是"模块化框架"——想加功能？插一个新 Agent 就行。这个架构的核心优势是：你以后的需求变化不需要重写整个系统。

---

### 2.4 爬虫 Agent（酷狗热歌榜，单源）

#### 原文

**FR-0011 音乐热歌榜爬取**：爬虫 Agent 抓取热歌榜数据。数据字段：歌曲名、歌手、排名、热度值、封面 URL、爬取时间。单次爬取成功率 ≥ 95%，去重入库，异常重试 3 次。手动触发，最小间隔 10 分钟。

> 注：SRS 中基础版数据源为 QQ 音乐，报价方案确认为基础版爬取酷狗热歌榜。以报价方案为准。

---

#### 怎么讲

**干什么**：一个自动"数据采集员"。你点一下按钮，它就去酷狗热歌榜把前 100 首歌的名字、歌手、排名、热度全扒下来，存到数据库里。同一首歌不会重复存，失败了自动重试 3 次。

**通俗类比**：就像你雇了个实习生，每天早上帮你看榜单、抄到 Excel 里。区别是：实习生要工资，可能抄错，还累；这个 Agent 一键搞定，0 出错率。

**为什么比竞品好**：竞品要么没有自动采集，数据得手动录入；要么采集完不会去重，数据库里一堆重复垃圾。我们是专业爬虫架构——成功率 95%+、自动去重、失败重试、异常日志，一条龙。

---

### 2.5 分析 Agent（基础风格分类，5 类）

#### 原文

**FR-0014 基础风格分类**：分析 Agent 对爬取的音乐数据进行风格分类。类别：流行 (Pop)、摇滚 (Rock)、电子 (Electronic)、民谣 (Folk)、R&B。分类置信度 ≥ 0.6 标记为有效。支持单首和批量分析。结果包含风格标签 + 置信度分数。

---

#### 怎么讲

**干什么**：一个自动"乐评人"。把爬下来的歌丢给它，它告诉你每首歌是什么风格、判断有多靠谱（比如"85% 把握这是流行"）。

**通俗类比**：就像去唱片店，店员能快速告诉你"这张是摇滚区、那张是电子区"。我们这个是 AI 版店员，一秒分析几百首，还给你置信度——不会硬装懂，不确定的标出来。

**为什么比竞品好**：竞品的风格标签大多是"人工打标"——几十首歌还好，几百首上千首呢？我们的 Agent 是全自动的，而且输出置信度，不是拍脑袋说"这是流行"就完了，数据可追溯。

---

### 2.6 创作 Agent（对接外部创作网站）

#### 原文

**FR-0018 外部创作网站对接**：创作 Agent 通过 Web 自动化操控外部在线音乐创作工具。用户提交创作参数（风格、BPM、调性、情绪描述），Agent 自动执行。任务状态实时反馈（排队中 → 执行中 → 完成/失败）。产物（MIDI/音频链接）保存可预览。

**FR-0019 创作任务管理**：可查看、取消、重试创作任务。历史任务保留 30 天。

---

#### 怎么讲

**干什么**：一个能替你操作创作软件的"机械手"。你告诉它"给我做一首 BPM 120 的电子乐，偏激昂"，它就去打开在线创作工具，按你的参数自动操作，做完把结果链接给你。

**通俗类比**：就像你请了一个录音棚助手——你跟他说"我要这个风格这个速度"，他帮你调设备、操作软件、导出文件，你只需要听结果、给意见。你不用学怎么操作那个复杂的 DAW 软件，把创作意图告诉 Agent 就行。

**为什么比竞品好**：竞品要么内置一个简陋的编辑器（功能远不如专业工具），要么让你自己去外面做完了再导回来。我们是"桥接模式"——对接的是市面上专业的在线创作工具，你拿到的是专业级产出，操作却是傻瓜级的。

---

### 2.7 Dashboard（统计卡片 + 基础表格）

#### 原文

**FR-0020 统计卡片**：首页展示今日爬取歌曲数、分析完成数、创作任务数、活跃 Agent 数。数据每 30 秒自动刷新，卡片点击可跳转详情。

**FR-0021 基础数据表格**：以表格形式展示爬取数据、分析结果、任务列表。分页默认 20 条/页，支持按列排序，支持关键字搜索过滤。

---

#### 怎么讲

**干什么**：平台的"仪表盘"。一登录就看到四张数字卡片——今天扒了多少歌、分析了几首、创作任务几个在跑、几个 Agent 活着。下面是个大表格，所有数据都能翻页、排序、搜索。

**通俗类比**：就像开车时的仪表盘——速度、油量、里程一目了然，不用打开引擎盖看。我们这个是平台的仪表盘，所有关键数据一屏看完。

**为什么比竞品好**：竞品很多没有统一 Dashboard，数据散落在不同页面，想看个统计得来回切。我们是集中式看板，30 秒自动刷新，数据实时性有保障。

## 三、8K 版新增 9 项

### 3.1 多源爬虫 + 定时调度

#### 原文

**FR-0012 多源爬取【8K】**：爬虫 Agent 支持网易云音乐和酷狗音乐多数据源爬取。多源数据统一格式存储，来源字段区分。成功率各 ≥ 90%。

**FR-0013 定时调度【8K】**：支持 Cron 表达式配置定时爬取任务。默认每日 02:00 自动执行全量爬取。同一数据源同时仅允许一个爬取任务运行。

---

#### 怎么讲

**干什么**：基础版只扒酷狗一家，8K 版再接入网易云和 QQ 音乐，三家的热歌榜全收。而且不用手动点，设置好时间它每天凌晨 2 点自动跑——你早上来上班，昨晚的榜单数据已经整整齐齐在数据库里了。

**通俗类比**：基础版是雇了 1 个实习生盯 1 个网站；8K 版是雇了 3 个实习生盯 3 个网站，而且不用你催，到点自己干活。

---

### 3.2 分析增强（情绪 / 编曲 / 和弦）

#### 原文

**FR-0015 情绪分析【8K】**：分析 Agent 对音乐进行情绪维度分析。维度包括愉悦度 (Valence)、唤醒度 (Arousal)、能量水平，每个输出 0.0-1.0 归一化分数。结合三值输出情绪标签（如"激昂"、"舒缓"、"忧郁"）。

**FR-0016 编曲分析【8K】**：分析 Agent 对音乐编曲结构进行分析。维度：BPM 估算（误差 ≤ ±5）、调性识别（准确率 ≥ 80%）、乐器组成推测、段落结构识别。结果以结构化 JSON 输出。

**FR-0017 和弦分析【8K】**：分析 Agent 对音乐和弦进行识别。输出和弦进程序列（如 I-V-vi-IV），识别常用和弦进行模式。结果可与创作 Agent 联动。

---

#### 怎么讲

**干什么**：基础版只告诉你"这歌是流行"；8K 版告诉你——
- 情绪：这歌是开心的还是丧的？紧张还是放松？能量高还是低？（愉悦度 0.8 + 唤醒度 0.6 = "激昂"）
- 编曲：BPM 多少？什么调？用了什么乐器？主歌副歌怎么分的？
- 和弦：什么和弦进行？是不是经典的 I-V-vi-IV（全世界一半的流行歌走的这个套路）？

**通俗类比**：基础版像"大众点评 3 星"，8K 版像"米其林评审报告"——色香味、摆盘、食材、技法全给你拆开了讲。

---

### 3.3 音乐在线试听

#### 原文

**FR-0028 音乐在线试听【8K】**：生成的音乐可直接在网页端播放试听。支持 Web Audio API 在线播放，含播放/暂停/进度控制。支持 MP3/WAV 格式。播放列表与创作任务结果联动。

> 基础版仅提供文件名列表，不可在线播放。

---

#### 怎么讲

**干什么**：基础版创作完只给你一个文件列表，你得下载到本地用播放器听；8K 版直接在网页上点播放按钮就能听，播放/暂停/拖进度条都有。

**通俗类比**：基础版像"寄 CD 给你"；8K 版像"网易云音乐网页版"——在线点开就听，不用下载。

---

### 3.4 风格对比面板

#### 原文

**FR-0029 风格对比面板【8K】**：多首歌曲的风格特征并排对比，ECharts 雷达图可视化。支持 2-6 首歌曲同时对比，雷达图展示风格/情绪/能量等多维度。对比结果支持截图导出。

> 基础版仅支持单首歌曲查看。

---

#### 怎么讲

**干什么**：基础版只能一首一首看分析结果；8K 版支持把 2 到 6 首歌放在一个雷达图里对比——谁的 BPM 更快、谁的情绪更激昂、谁的和弦更复杂，一张图一目了然。

**通俗类比**：基础版像"一张一张翻履历"；8K 版像"把 6 个候选人排成一排同时面试"——高下立判。

---

### 3.5 创作历史与版本管理

#### 原文

**版本管理【8K】**（来源：报价方案 3.2）：每首歌保留多个版本，随时回退切换。版本快照，支持预览/对比/回退。

> 基础版仅保留最新一版。

---

#### 怎么讲

**干什么**：基础版生成一首歌，改了就覆盖，旧版本没了；8K 版每首歌保留多个版本——v1、v2、v3……随时回退、对比，不怕改坏了。

**通俗类比**：基础版像"Word 不保存直接关"；8K 版像"Google Docs 版本历史"——任何时候都能回到昨天下午 3 点的那个版本。

---

### 3.6 系统备份与回档

#### 原文

**FR-0023 定时备份【8K】**：PostgreSQL 数据库定时全量备份。默认每日 03:00 自动备份，AES-256 加密存储，保留最近 30 天自动清理。

**FR-0024 手动备份【8K】**：管理员可手动触发即时备份，完成后提供下载链接。

**FR-0025 一键回档【8K】**：管理员选择历史备份执行回档。回档前自动生成当前快照备份，过程进入维护模式（拒绝新请求），全程 ≤ 10 分钟。需二次确认并记录操作日志。

---

#### 怎么讲

**干什么**：每天凌晨 3 点自动把整个数据库加密备份一份，保留 30 天。万一哪天数据出问题了（误删、中毒、操作失误），管理员点一下"回档"，10 分钟内恢复到出问题前的状态。

**通俗类比**：就像 iPhone 的 iCloud 自动备份——你平时感觉不到，但手机坏了换新机，所有数据一键回来。AES-256 加密意味着备份文件本身就是锁着的，别人拿到了也打不开。

---

### 3.7 操作日志

#### 原文

**FR-0026 全量操作记录【8K】**：记录所有用户的增删改操作。字段：操作人 ID、操作类型、目标资源、操作详情 (JSON)、IP 地址、时间戳。日志写入异步，不影响主业务。按月分表，保留 180 天。

**FR-0027 审计查询【8K】**：管理员可按操作人、操作类型、时间范围筛选日志。支持关键字搜索操作详情。查询结果可导出 CSV。

---

#### 怎么讲

**干什么**：平台上谁干了什么，全都记下来。谁几点登录、谁删了什么数据、谁改了谁的权限——可查、可追溯、可导出 Excel。保留半年。

**通俗类比**：就像大厦的监控摄像头——平时不用看，但出了事你能回放"那天下午谁进了这个房间"。

---

### 3.8 热插拔 5 槽位

#### 原文

**FR-0007 Agent 槽位【8K】**：完整版支持最多 5 个槽位。

**FR-0009 Agent 热插拔【8K】**：Agent 运行中动态加载与卸载，无需重启系统。加载后 10 秒内完成注册，卸载时优雅关闭（等待当前任务完成或超时 30s 强制终止）。操作不影响其他 Agent。事件记录到操作日志。

---

#### 怎么讲

**干什么**：基础版 3 个槽位 + 手动开关；8K 版 5 个槽位 + "热插拔"——你想加一个 Agent，直接在管理后台插上，10 秒内它就上线干活了，其他 Agent 不受影响。不用停服、不用重启。

**通俗类比**：基础版像台式机要关机才能插拔显卡；8K 版像 USB——插上就用，拔掉就走，电脑照常运行。

### 3.9 ECharts 可视化图表

#### 原文

**FR-0022 ECharts 可视化图表【8K】**：Dashboard 集成 ECharts，提供多维度数据可视化。图表类型：饼图（风格分布占比）、柱状图（各数据源歌曲数量对比）、趋势线（爬取量/分析量时间趋势）、热力图（情绪维度分布）。支持时间范围筛选（7天/30天/自定义），数据与后端 API 实时同步，图表可导出为 PNG 图片。

#### 怎么讲

**干什么**：基础版只有数字卡片 + 表格；8K 版的 Dashboard 有了真正的图表——饼图看风格占比、柱状图看数据源对比、趋势线看增长曲线、热力图看情绪分布。所有图表支持导出图片，直接放进报告里。

**通俗类比**：基础版像"算盘记账"——数字都在，但得自己看；8K 版像"Tableau 商业智能大屏"——数据变成图表，趋势一眼看清，老板看了也点头。

## 七、客户三问——标准回答

### Q1：竞品也有 AI 分析功能，你们有什么不一样的？

**推荐回答**：竞品的 AI 能力是"写死的"——分析什么、怎么分析，出厂就定了。我们用的是 Agent 热插拔框架，分析能力是"可插拔的模块"。你将来想让系统学会分析"音乐中的民族元素占比"？写一个新 Agent 插上去就行，不需要重做系统。这个架构给了你持续扩展的可能性，而不是买完就定型了。

另外竞品大多是"一体机"——数据采集、分析、创作要么没有，要么功能割裂。我们是从爬数据 → 分析 → 辅助创作，一条完整链路，数据在一个平台上流转，不需要导出导入来回折腾。

---

### Q2：基础版和 8K 版差了 2000 块，值不值？

**推荐回答**：2000 块的差价买的是 9 项企业级能力，逐项拆开看：

- 多源爬虫 + 定时调度：如果不加，你每天得手动点按钮，只能看酷狗一家。加了之后全自动 + 三平台覆盖，省的是你每天的人工时间。
- 情绪/编曲/和弦分析：基础版只告诉你"这是流行歌"，8K 版告诉你 BPM、调性、和弦进行——如果你要正经做音乐分析，这是必需品。
- 备份回档：没有这一项，数据丢了就真没了。有了就是每天自动加密备份 + 一键恢复，这是保险。
- 操作日志：团队用的必备项——谁动了什么数据，有据可查。
- 在线试听 + 对比面板 + 版本管理 + ECharts 图表：每一项都是体验升级，让你从"能用"变成"好用"。

说实话，如果你只是用一两个月玩玩，基础版够了。如果你是长期用、团队用、要把数据当资产，8K 版里备份回档和操作日志这两项就值回差价了。

---

### Q3：交付后我自己想加功能怎么办？

**推荐回答**：三个途径。

第一，Agent 热插拔框架本身就是为扩展设计的——如果你想接入一个新的数据源或者加一个新的分析维度，可以找我们开发一个新 Agent，插上去就能用，定价按工作量评估。

第二，你有自己的技术团队的话，Agent 接口是标准化的，你的工程师可以按照接口规范自己开发，不需要动核心系统。

第三，3 个月免费维护期内出 bug 我们负责修。维护期过后要加新功能，另报就行，按人天或者按模块报价。

我们不想把你锁死，平台设计之初就考虑了"客户将来会有新想法"这个事实。
