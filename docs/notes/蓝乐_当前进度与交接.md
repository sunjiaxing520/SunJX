## 3. 项目一句话

蓝乐是一个 AI 音乐创作工作台。

它不是自研音乐生成模型，而是把榜单数据、音乐分析、AI 作词和 Suno 创作串成一个可操作、可演示、可验收的平台。

首期业务链路：

```text
登录 -> 酷狗榜单采集 -> 音乐分析 -> AI 作词 -> Suno 创作 -> Dashboard 展示
```

## 4. 首期边界

首期做：

```text
人操作多个固定 Agent
```

首期不做：

- 用户自定义工作链。
- 总调度 Agent 自动拆任务。
- 多 Agent 自动协作。
- 复杂长期记忆系统。
- 多音乐平台数据源。
- 自研音乐生成模型。

这些作为后续增强和加费项目。

## 5. 四个核心 Agent

| Agent | 首期职责 |
|---|---|
| 酷狗爬虫 Agent | 采集酷狗热歌榜数据 |
| 音乐分析 Agent | 分析歌曲风格、情绪、关键词、节奏倾向 |
| 作词 Agent | 根据主题、风格、用户偏好生成和修改歌词 |
| Suno 创作 Agent | 整理素材，提交 Suno，跟踪状态并保存结果 |

## 7. 已完成的带教学习

已经带用户完成：

### 第 01 课：健康检查请求流程

学习链路：

```text
浏览器或测试客户端
-> backend/app/main.py
-> backend/app/api/v1/router.py
-> backend/app/api/v1/routes/health.py
-> backend/app/schemas/health.py
-> backend/app/services/health.py
-> 返回 JSON
```

用户需要理解：

- `main.py` 创建 FastAPI app。
- `router.py` 汇总接口模块。
- `routes/` 放具体接口。
- `schemas/` 定义请求和响应格式。
- `services/` 放业务逻辑。
- 测试可以用 `dependency_overrides` 替换数据库依赖。
