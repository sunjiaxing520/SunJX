## 7. 已讲过的关键概念

后端请求大致路径：

```text
浏览器 -> main.py -> api/v1/router.py -> routes/health.py -> schemas/services -> 返回 JSON
```

文件职责：

- `main.py`：创建后端 app，是后端入口。
- `api/v1/router.py`：汇总 v1 版本下的小路由。
- `routes/`：具体接口文件。
- `schemas/`：定义请求和返回 JSON 的格式。
- `services/`：放业务逻辑，例如判断数据库是否可用。
- `Depends`：FastAPI 的依赖注入，用来把数据库会话等对象交给接口。
- `dependency_overrides`：测试时用来替换真实依赖，避免测试连接真实数据库或外部服务。

用户明确要求：不要只说一堆术语，要按工程师真实开发流程教。

每一步都应该按这个闭环走：

1. 这一步要解决什么问题。
2. 为什么现在做这一步。
3. 会涉及哪些文件。
4. 命令是什么意思。
5. 输出结果怎么看。
6. 改完如何测试。
7. 测试结果说明了什么。
8. 是否需要提交和推送。

一次只讲透一个小闭环，不要一次塞太多抽象概念。
